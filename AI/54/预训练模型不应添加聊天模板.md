# 预训练模型不应添加聊天模板

### 🧑‍💻 User

llm的pretrain base model 应该考虑添加jinja chat template吗？

---

### 🤖 Assistant

在 pretrain base model 阶段，**通常不应该添加 chat template**，但建议在**发布模型时包含它**。让我详细解释原因：

## 为什么 Pretrain 阶段不需要

### 1. **训练目标不同**
```python
# Pretrain 阶段只学习语言建模
# 模型看到的只是连续的文本
text = "The capital of France is Paris. It is known for..."

# 而不是对话格式
# <|user|>What is the capital of France?<|assistant|>Paris<|end|>
```

### 2. **数据格式不匹配**
Pretrain 数据通常是：
- 书籍、网页、论文等自然文本
- 没有对话结构标记
- 没有角色区分

添加 chat template 会让模型学习到错误的格式期望。

## 何时应该添加

### 在 SFT/Instruction Tuning 阶段
```json
{
  "messages": [
    {"role": "system", "content": "You are helpful."},
    {"role": "user", "content": "What is AI?"}
  ],
  "template": "{% for message in messages %}{{'<|' + message['role'] + '|>' + message['content'] + '<|end|>'}}{% endfor %}"
}
```

## 最佳实践建议

<details>
<summary><strong>✅ 推荐做法</strong></summary>

### 1. Base Model 发布时：
```python
# tokenizer_config.json
{
  "chat_template": null,  # 或者完全省略
  "model_type": "llama",
  "bos_token": "<s>",
  "eos_token": "</s>"
}
```

### 2. Chat/Instruct Model 发布时：
```python
# tokenizer_config.json
{
  "chat_template": "{% for message in messages %}{% if message['role'] == 'system' %}{{ '<|system|>' + message['content'] + '<|end|>' }}{% elif message['role'] == 'user' %}{{ '<|user|>' + message['content'] + '<|end|>' }}{% elif message['role'] == 'assistant' %}{{ '<|assistant|>' + message['content'] + '<|end|>' }}{% endif %}{% endfor %}{% if add_generation_prompt %}{{ '<|assistant|>' }}{% endif %}"
}
```

</details>

### 3. 使用 HuggingFace 时的处理：
```python
from transformers import AutoTokenizer

# Base model - 手动处理格式
tokenizer = AutoTokenizer.from_pretrained("model/base")
text = "Q: What is AI?\nA:"
inputs = tokenizer(text, return_tensors="pt")

# Chat model - 使用 template
tokenizer = AutoTokenizer.from_pretrained("model/chat")
messages = [{"role": "user", "content": "What is AI?"}]
inputs = tokenizer.apply_chat_template(messages, return_tensors="pt")
```

## 注意事项

⚠️ **如果错误地添加 chat template 到 base model：**
- 用户可能误用 `apply_chat_template()` 得到格式错误的输入
- 模型可能在继续预训练时学到错误的模式
- SFT 阶段需要重新配置，造成混淆

🎯 **核心原则**：Base model 应该保持"纯净"，只包含基础 tokenizer 配置；Chat template 是下游任务的特定配置，应该在 instruction tuning 阶段引入。

### 🤖 Assistant

很好的观察！根据 Frontier AI Lab 的最新实践（特别是 Llama 3、Qwen 2.5 等模型的技术报告），**现代预训练中混入 instruct 数据主要有两种方式**：

## 1. 全预训练阶段混合（Proportional Mixing）

```python
# 整个预训练过程中按比例混合
pretrain_data = {
    "web_text": 0.70,      # 70% 网页文本
    "code": 0.10,          # 10% 代码
    "math": 0.08,          # 8% 数学
    "instruct_like": 0.05, # 5% 指令式数据
    "synthetic": 0.07      # 7% 合成数据
}
```

**典型案例：**
- **Llama 3** 在整个预训练中混合了约 5% 的高质量 instruct 数据
- **Qwen 2.5** 在预训练中加入了 conversational data

## 2. 退火阶段添加（Annealing Phase Mixing）

```python
# 训练进度
total_steps = 1_000_000
annealing_start = 950_000  # 最后 5% 的步数

def get_training_data(step):
    if step < annealing_start:
        # 正常预训练数据
        return standard_pretrain_data()
    else:
        # 退火阶段：大幅提高高质量数据比例
        return {
            "web_text": 0.20,
            "code": 0.15,
            "math": 0.15,
            "instruct_data": 0.30,  # 显著增加
            "synthetic_qa": 0.20
        }
```

## 实际案例分析

<details>
<summary><strong>📊 主流模型的具体做法</strong></summary>

| 模型 | 方法 | Instruct数据占比 | 时机 |
|------|------|-----------------|------|
| **Llama 3** | 全程混合 | ~5% | 全训练周期 |
| **Llama 3.1** | 退火增强 | 10-15% | 最后10%步数 |
| **Qwen 2.5** | 混合+退火 | 3-5%→15% | 全程+末段增强 |
| **DeepSeek V2** | 多阶段退火 | 逐渐增加 | 分阶段增强 |
| **Phi-3** | 合成数据混合 | 10-20% | 全周期 |

</details>

## 为什么选择不同的策略

### 全程混合的优势：
```python
# 模型从始至终学习多种数据分布
# 优点：更平滑的分布学习
# 缺点：可能影响长文本建模能力
def full_mixing():
    for step in range(total_steps):
        data = sample_mixed_distribution()  # 固定比例
        loss = model(data)
```

### 退火阶段加入的优势：
```python
# 先学习基础能力，再注入特定能力
# 优点：保持强语言建模能力，同时获得指令遵循
# 缺点：需要精心设计退火策略
def annealing_mixing():
    for step in range(total_steps):
        if step < 800_000:
            data = standard_data()  # 纯预训练
        elif step < 900_000:
            data = mixed_data(instruct_ratio=0.1)
        else:
            data = mixed_data(instruct_ratio=0.3)
        loss = model(data)
```

## 关键洞察

🔑 **现代实践的核心变化：**

1. **"Base" 和 "Instruct" 的边界正在模糊**
   ```python
   # 传统认知：
   Base Model = 纯语言建模
   Instruct Model = Base + SFT + RLHF
 
   # 现代实践：
   Modern Base = 语言建模 + 少量instruct + 合成数据
   Modern Instruct = Modern Base + 高质量SFT + 偏好优化
   ```

2. **数据质量 > 数据格式**
   - 即使预训练中包含对话格式，只要数据质量高，也能提升模型能力
   - 关键在于数据的 informative 程度，而非格式

3. **Chat template 的实际作用**
   ```python
   # Chat template 更像是一个"接口规范"
   # 而不是区分 base/instruct 的根本标志
 
   # Base model 可能已经见过类似格式：
   "Question: What is 2+2?\nAnswer: 4"
 
   # Instruct model 使用标准化的：
   "<|user|>What is 2+2?<|assistant|>4<|end|>"
   ```

## 实践建议

如果你在训练自己的模型：

```python
# 1. 预训练阶段可以适当加入结构化数据
pretraining_mix = {
    "raw_text": 0.70,
    "structured_qa": 0.10,  # 简单的QA格式
    "code": 0.15,
    "math": 0.05
}

# 2. 退火阶段增强instruct能力
def annealing_data_loader(current_step, total_steps):
    progress = current_step / total_steps
    if progress > 0.9:
        instruct_ratio = 0.2 + (progress - 0.9) * 3  # 20% -> 50%
        return get_mixed_data(instruct_ratio=instruct_ratio)
    return standard_data

# 3. 最后在SFT时才添加标准的chat template
```

✨ **结论**：现代预训练确实混入了 instruct 数据，这既包括全程混合（保持小比例），也包括退火阶段特意增加比例。但这并不意味着要在 base model 中添加 chat template——template 仍然是 SFT 阶段的产物，只是 base model 可能已经对结构化对话格式不陌生了。
