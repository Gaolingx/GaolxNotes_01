# 随机初始化Transformer学不动原因

### 🧑‍💻 User

为什么完全随机初始化的transformers用celoss+小lr（例如<1e-5）根本学不动？这和激活值/特征值/优化动力学/损失曲面有什么关系

---

### 🤖 Assistant

完全随机初始化的 Transformer 在面对交叉熵损失（Cross-Entropy Loss）时，如果使用极小的学习率（例如 $< 1e-5$ 且没有 Warmup），几乎必然会陷入**“学不动”（训练停滞）**的状态。

这是一个非常经典的深度学习优化问题。导致这一现象的核心原因可以从**激活值、特征值、优化动力学和损失曲面**四个维度来系统拆解。

以下是具体的深度剖析：

<details open>
<summary><b>1. 激活值视角：注意力崩溃（Attention Collapse）与过度平滑</b></summary>

在完全随机初始化时，Transformer 的注意力机制处于一种高度对称且低效的“平均”状态。

*   **均匀的注意力矩阵：** 在初始化阶段，Query 和 Key 矩阵是随机的。对于任意两个 token $i$ 和 $j$，它们的内积 $q_i \cdot k_j / \sqrt{d}$ 期望接近于 0。经过 Softmax 后，注意力权重矩阵几乎是一个均匀分布的矩阵（每个值约等于 $1/L$，其中 $L$ 是序列长度）。
*   **过度平滑（Oversmoothing）：** 因为注意力权重是均匀的，Self-Attention 层的输出变成了所有输入 token 向量的**简单全局平均**。随着层数加深，不同 token 的激活值会变得极其相似。
*   **小 LR 的致命影响：** 要打破这种“均匀注意力”的对称性，模型需要通过梯度更新来拉大 $Q$ 和 $K$ 的差异，使得 Softmax 能够产生尖锐的（sharp）注意力分布。如果学习率 $< 1e-5$，这种微小的更新量在经过 Softmax 的指数平滑后，根本无法引起注意力权重的实质性改变。模型被永远锁死在“输出全局平均值”的状态。

</details>

<details open>
<summary><b>2. 优化动力学视角：极低的梯度信噪比（SNR）与 CE Loss 的特性</b></summary>

交叉熵（CE）损失函数在初始化阶段结合小学习率，会产生极其糟糕的优化动力学。

*   **初始的概率分布：** 随机初始化下，模型输出的 Logits 分布也是随机的。经过 Softmax 后，模型对词表中所有词的预测概率 $p_i$ 约等于 $1/V$（$V$ 为词表大小，通常在 $30,000$ 以上）。此时的 CE Loss 非常大，接近 $\log(V)$。
*   **梯度的方向冲突：** CE Loss 对 Logit $z_i$ 的梯度是 $\frac{\partial L}{\partial z_i} = p_i - y_i$（$y_i$ 是 One-hot 标签）。在初始化时，对于正确的类别，梯度是 $1/V - 1 \approx -1$；对于大量错误的类别，梯度是 $1/V \approx 0$。
*   **信噪比（SNR）极低：** 在一个 Batch 中，每个样本都在试图把正确的 token 的 Logit 往上拉。由于特征被高度平均化（见第一点），不同样本对应的梯度方向在参数空间中是**高度随机且相互抵消的**。这就导致了梯度的方差极大，但均值（真实下降方向）极小。
*   **Adam 优化器的失效：** Transformer 通常使用 Adam。Adam 会根据梯度的二阶矩（方差）来归一化更新步长。当梯度充满噪声且学习率 $< 1e-5$ 时，参数的更新量 $\Delta \theta$ 变成了纯粹的随机游走（Random Walk）。这种微弱的随机游走根本无法积累起足够的动量（Momentum）来穿越噪声区。

</details>

<details open>
<summary><b>3. 损失曲面视角：广阔的高维鞍点平原（Saddle Plateaus）</b></summary>

Transformer 的损失曲面（Loss Landscape）在初始化点附近是出了名的恶劣。

*   **平原现象：** 在随机初始化点，由于注意力机制的均匀性，无论输入什么，输出都差不多。这意味着在这个区域内，稍微改变一点权重，Loss 几乎不会变化。在损失曲面上，这表现为一个广阔平坦的高维平原或鞍点区域。
*   **缺乏 Warmup 的惩罚：** 为什么 Transformer 通常需要较大的学习率加上 **Linear Warmup**？Warmup 的本质就是在训练初期，通过逐渐增大的步长，强行让参数“冲出”这片初始的平原区，建立起初步的特征表达能力，到达一个曲率更正常、梯度指向更明确的盆地（Basin）。
*   **被困于平原：** 当学习率固定在 $< 1e-5$ 时，优化的步长太短。模型相当于在一个一望无际的平原上以蜗牛的速度蠕动，由于周边的 Loss 都是 $\log(V)$，梯度很小且充满噪声，优化过程会提前停滞，陷入“假收敛”。

</details>

<details open>
<summary><b>4. 特征值视角：表示坍塌（Representation Collapse / Anisotropy）</b></summary>

这是从表征空间（Embedding Space）的角度来看为什么学不动。

*   **各向异性（Anisotropy）：** 随机初始化的 Transformer 在前向传播时，特别是在 Post-LayerNorm 架构下，特征向量（Token Representations）倾向于退化到一个狭窄的锥形空间内。这被称为表示坍塌。
*   **无法区分的特征：** 当所有 token 的特征在经过几层之后都挤在一起时，最后的线性分类器（LM Head）根本无法将它们线性可分。
*   **微小 LR 无法重塑空间：** 要解决表示坍塌，需要对底层 Embedding 和 MLP 层的权重进行大幅度的更新，以“撑开”特征空间，增加特征的方差。小于 $1e-5$ 的学习率产生的更新量，远远不足以抵抗 LayerNorm 和残差连接带来的特征汇聚效应。

</details>

### 总结

完全随机初始化的 Transformer 加上 $< 1e-5$ 的微小学习率之所以“学不动”，是因为：

1.  **注意力机制**处于全局平均的对称状态（Attention Collapse）；
2.  **特征空间**处于高度重合的坍塌状态（Representation Collapse）；
3.  **CE Loss** 在此时产生的梯度方向极其混乱，信噪比极低；
4.  微小的学习率无法打破对称性，也无法积累足够的动量让模型逃离初始的**高维损失平原**。

**解决方案：** 这就是为什么 Transformer 的标准训练范式必须包含 **参数初始化技巧（如截断正态分布、缩放初始化）** 以及 **Learning Rate Warmup（通常峰值 LR 至少在 $1e-4$ 到 $1e-3$ 级别）**。Warmup 提供的大步长是打破初始对称性、逃离平原的必须“启动资金”。

### 🧑‍💻 User

既然需要大lr迫使优化器逃离鞍点，那为什么预训练阶段初期还需要一个几百 steps左右的lr warmup stage？

---

### 🤖 Assistant

这是一个极其敏锐的直觉！这看起来确实像一个悖论：**既然我们认定初始状态是一个需要被打破的平原/鞍点，且小学习率（LR）会导致停滞，那为什么不一上来直接用最大的 LR“猛踹”模型一脚，而是要用几百到几千步的 Warmup 从零慢慢爬升？**

核心原因可以用一句话概括：**我们需要大 LR 来“走得远”，但在初始阶段，梯度的方向是纯粹的“噪音”，如果一上来就盲目跨出大步，模型不但无法安全逃离平原，反而会直接“坠入深渊”（导致数值爆炸或永久性的梯度消失）。**

Warmup 并不是为了“放慢”逃离的速度，而是为了在“踩死油门”之前，先让优化器**校准方向并建立牵引力**。具体原因可以从以下四个维度深入理解：

<details open>
<summary><b>1. 优化器动力学：Adam 的“冷启动”危机（Cold Start Problem）</b></summary>

Transformer 几乎全部使用 Adam 或 AdamW 优化器。Adam 的核心思想是利用梯度的滑动平均值（一阶矩 $m_t$）和方差（二阶矩 $v_t$）来动态调整每个参数的步长：
$$ \theta_{t} = \theta_{t-1} - \eta \frac{m_t}{\sqrt{v_t} + \epsilon} $$

*   **初始的方差估计极度不准：** 在前几个 step，由于没有历史信息，$v_t$ 完全由当前 batch 产生的极端嘈杂的梯度决定。
*   **如果不加 Warmup（突发大步长）：** 如果在 step 1 就使用峰值 LR（例如 $1e-3$），由于初始梯度的方差很大且不稳定，某些参数的分母 $\sqrt{v_t}$ 可能很小，导致该方向上的实际更新步长被异常放大。参数会发生剧烈的、方向随机的跳跃。
*   **Warmup 的作用：** Warmup 阶段的极小 LR 限制了初始几步的实际更新幅度。它允许 Adam 在权重几乎不发生破坏性改变的情况下，静静地“观察”几百个 batch，积累足够多真实的梯度样本，算出稳定可靠的 $m_t$ 和 $v_t$。当 LR 真正升到最大值时，Adam 已经知道哪些方向是噪声，哪些方向是真正的下降通道了。

</details>

<details open>
<summary><b>2. 激活值视角：防止 Softmax 瞬间饱和与“永久性死亡”</b></summary>

我们在上一个问题提到，要打破 Attention 的均匀状态需要拉大 Query 和 Key 的差异。但这个“拉大”必须是温和的。

*   **Softmax 的脆弱性：** 注意力权重和最终的预测概率都经过 Softmax 函数：$p_i = \frac{\exp(z_i)}{\sum \exp(z_j)}$。
*   **如果不加 Warmup（突发大步长）：** 巨大的初始 LR 会在一两步之内，给 $Q$、$K$ 矩阵或最后的 LM Head 权重施加一个极大的随机更新量。这会导致某些 Logit $z_i$ 的绝对值瞬间飙升（比如变成 $+100$ 或 $-100$）。
*   **梯度消失的深渊：** 一旦 $z_i$ 变得极大，Softmax 的输出就会变成极端的 $1.0$ 或 $0.0$。此时，Softmax 的局部梯度 $\frac{\partial p_i}{\partial z_j} = p_i(\delta_{ij} - p_j)$ 将完全变成 $0$。**模型从“鞍点平原”直接跳进了一个梯度为 0 的“死坑”**。由于梯度没了，即使后面有再好的数据，这些参数也永远无法被更新回来（这被称为 Attention Death 或 Dead Logits）。
*   **Warmup 的作用：** 让模型在不触发 Softmax 饱和的前提下，缓慢地打破对称性，安全地过渡到有意义的特征表示。

</details>

<details open>
<summary><b>3. 架构视角：深层网络的梯度放大效应（特别是 Post-LN）</b></summary>

早期 Transformer（如原始的 Attention is All You Need）使用的是 Post-LayerNorm 架构。

*   **深层梯度的剧烈累积：** 在 Post-LN 架构中，靠近输入底层的参数在初始化阶段接收到的梯度，其方差是顶层梯度的数倍甚至数十倍（随着层数 $L$ 的增加而急剧放大）。
*   **如果不加 Warmup：** 顶层的更新可能刚刚好，但底层的 Embedding 和初期 Attention 层的权重会被这股巨大的梯度瞬间摧毁，导致整个网络的前向传播在第二步就变成 NaN（数值溢出）。
*   *补充说明：* 虽然现代大模型大多改用 Pre-LayerNorm 或 RMSNorm，缓解了底层梯度爆炸的问题，使得网络对 Warmup 的依赖度有所降低，但由于上述的 Adam 冷启动和 Softmax 饱和问题依然存在，Warmup 仍然是必选项。

</details>

<details open>
<summary><b>4. 损失曲面视角：寻找平原上的“出口”</b></summary>

初始的损失曲面是一个高维的平原（Plateau），但这个平原的边缘并不是平缓的下坡，而是布满了陡峭的峡谷和悬崖。

*   **如果不加 Warmup：** 一开始就全速奔跑（大 LR），由于初始的梯度方向是各个 batch 数据的随机博弈（信噪比极低），模型极大概率会一头撞在“悬崖”上，导致 Loss 瞬间飞升（Spike）。
*   **Warmup 的作用：** 我们可以把 Warmup 看作是在黑暗的平原上“打着手电筒慢慢摸索”。在最初的几百步里，模型整合了成千上万个样本的微小共识，缓慢地将权重移动到平原边缘一个真正平滑的“下坡通道”入口。一旦找到了正确的下坡方向，LR 也刚好 Warmup 到了峰值，此时再借着大 LR 呼啸而下，完成高效的优化。

</details>

### 总结比喻

把训练完全随机初始化的 Transformer 想象成**在冰面上驾驶一辆静止的跑车**。

*   上一个问题我们讨论了：你不能只给一点点油门（`< 1e-5` 的固定小 LR），那样车子根本动不了，只能在原地打滑（陷入鞍点，学不动）。我们需要极高的速度（大 LR）才能冲出这片冰面。
*   这个问题解释了：**但你绝对不能在静止状态下一脚把油门踩到底（无 Warmup，初始极大 LR）**。因为轮胎没有抓地力（Adam 的动量和方差未建立），方向盘也是乱打的（初始梯度方向是噪音）。如果一脚油门到底，跑车会瞬间失控、原地打转甚至翻车（Softmax 饱和、梯度消失、数值爆炸）。

**Warmup 就是起步时的“循序渐进”：先轻踩油门，让轮胎获得抓地力，修正好方向，然后再逐渐把油门踩到底，从而安全、高速地冲出初始的平原区。**
