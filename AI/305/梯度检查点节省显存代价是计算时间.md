# 梯度检查点节省显存代价是计算时间

### 🧑‍💻 User

如何理解Gradient Checkpointing，在训练像LLM这样参数量很大的参数时是如何节约大量的显存的，代价是什么？

---

### 🤖 Assistant

**Gradient Checkpointing**（梯度检查点，也常称为 Activation Checkpointing）是一种以**时间换空间**的显存优化技术。

在训练像 LLM（大语言模型）这样参数巨大的模型时，显存通常是最稀缺的资源。Gradient Checkpointing 通过**不保存部分中间激活值（Activations），而在反向传播时重新计算它们**，从而显著降低显存占用。

结合你提供的代码片段，我们可以深入理解它的原理、节省显存的机制以及代价。

---

### 1. 核心问题：为什么 LLM 训练耗显存？

在深度学习的标准训练流程中，显存主要被两部分占用：
1.  **模型参数（Model States）：** 包括权重、梯度和优化器状态（如 Adam 的动量）。
2.  **中间激活值（Activations）：** 前向传播（Forward Pass）过程中每一层产生的输出结果。

**对于 LLM，激活值占用的显存往往比参数本身还大。**
在标准的反向传播（Backpropagation）中，为了计算梯度（Gradient），我们必须保留**所有**层的输入和输出（激活值）直到反向传播结束。

*   **公式视角：** 假设模型有 $N$ 层，计算第 $i$ 层的梯度 $\frac{\partial L}{\partial W_i}$ 通常需要利用链式法则，依赖于该层的输入 $x_i$ 和传回来的误差 $\delta_{i+1}$。如果 $x_i$ 被丢弃了，就没法求导了。

### 2. Gradient Checkpointing 如何工作？

Gradient Checkpointing 的核心策略是：**不要保留所有层的激活值，只保留一部分关键节点的激活值（Checkpoints）。**

#### 过程对比：

*   **普通模式（无 Checkpointing）：**
    1.  **Forward:** $Layer_1 \rightarrow save(A_1) \rightarrow Layer_2 \rightarrow save(A_2) \rightarrow \dots \rightarrow Loss$
    2.  **Memory:** 内存中存满了 $A_1, A_2, \dots, A_N$。
    3.  **Backward:** 直接利用存好的 $A_i$ 计算梯度。

*   **Checkpointing 模式：**
    1.  **Forward:** $Layer_1 \rightarrow A_1 (\text{丢弃内部细节}) \rightarrow Layer_2 \rightarrow A_2 (\text{丢弃}) \dots$
        *   我们只保存被标记为 `checkpoint` 的那几个节点的输入（Input），丢弃层内部产生的庞大中间激活值。
    2.  **Memory:** 内存中只有少量的 Checkpoints（例如每隔几层存一个，或者每个 Transformer Block 的输入）。
    3.  **Backward:** 当反向传播到达第 $i$ 层需要用到激活值时，利用最近的 Checkpoint **重新运行一次前向传播（Re-forward）** 来临时生成所需的激活值，用完即扔。

### 3. 代码片段中的体现

在你提供的代码中，这一点体现得非常明显：

```python
# 你的代码片段
def forward_layers_checkpointed(
        self,
        xs: torch.Tensor,
        chunk_masks: torch.Tensor,
        pos_emb: torch.Tensor,
        mask_pad: torch.Tensor,
    ) -> torch.Tensor:
        for layer in self.encoders:
            # 关键点在这里：ckpt.checkpoint
            xs, chunk_masks, _, _ = ckpt.checkpoint(
                layer.__call__, xs, chunk_masks, pos_emb, mask_pad
            )
        return xs
```

*   **`ckpt.checkpoint` 的作用：**
    它告诉 PyTorch：“不要保存 `layer.__call__` 内部计算产生的所有中间张量（激活值）来用于反向传播。只把输入 `xs` 存下来。等到反向传播需要求导时，再把这个 `layer.__call__` 重新运行一遍。”
*   **对于 Transformer：** 通常会对每一个 Encoder/Decoder Layer 做 Checkpointing。这意味着显存中不再需要同时存放 100 层 Transformer 的所有 Attention Map 和 FFN 输出，只需要存放 100 个输入 Embedding 即可。

### 4. 显存节省了多少？

*   **数量级变化：**
    *   **不使用 Checkpointing：** 显存占用与层数 $N$ 成线性关系，即 $O(N)$。
    *   **使用 Checkpointing：** 如果配置得当（例如 $\sqrt{N}$ 策略），显存占用可以降低到 $O(\sqrt{N})$。对于像 GPT-3 或 LLaMA 这样深度的网络，这可以将激活值显存占用减少 **70% 到 90%**。
*   **实际意义：** 这使得我们可以在有限的 GPU 上训练 Batch Size 更大的数据，或者训练序列长度（Context Window）更长的模型。

### 5. 代价是什么？（The Cost）

天下没有免费的午餐，Gradient Checkpointing 的代价是**计算时间**。

1.  **计算开销增加（Compute Overhead）：**
    由于在反向传播时需要重新计算前向传播的部分逻辑，被 Checkpoint 标记的部分实际上执行了**两次**前向传播（一次是真正的 Forward，一次是 Backward 时的 Re-forward）。
2.  **训练速度变慢：**
    理论上，计算量增加了约 **33%**。
    *   标准：1 Forward + 1 Backward
    *   Checkpointing：1 Forward + 1 Re-Forward + 1 Backward
    *   (假设 Backward 的计算量大约是 Forward 的 2 倍，那么总时间从 $1+2=3$ 变成了 $1+1+2=4$，大约增加了 33%)。

### 总结

| 特性 | 标准训练 | Gradient Checkpointing |
| :--- | :--- | :--- |
| **显存占用 (VRAM)** | **极高** (存所有激活值) | **低** (只存少量关键点) |
| **计算量 (FLOPs)** | 标准 | 增加约 33% |
| **适合场景** | 小模型，或显存极其充裕 | **大模型 (LLM)**，长序列训练 |
| **原理** | 空间换时间 | **时间换空间** |

**一句话总结：**
Gradient Checkpointing 就像是在玩游戏时不每一步都存盘（保存所有显存），而是只在每一关的开头存个盘。如果你在关卡中间“挂了”（需要反向传播求导），你就读关卡开头的档，把这一关重打一遍（重计算）来恢复状态。虽然重打浪费了时间，但你不再需要存储每一秒的游戏状态，从而极大地节省了存储空间。
