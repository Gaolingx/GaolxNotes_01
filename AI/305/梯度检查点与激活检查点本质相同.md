# 梯度检查点与激活检查点本质相同

### 🧑‍💻 User

模型梯度检查点和激活检查点有什么区别？

---

### 🤖 Assistant

简而言之，在深度学习和大型语言模型（LLM）的训练中，**梯度检查点（Gradient Checkpointing）**和**激活检查点（Activation Checkpointing）**通常指的是**完全相同的技术**。它们只是同一概念在不同框架或文献中的不同叫法。

这项技术的核心思想是：**用额外的计算时间来换取显存（VRAM）空间。**

以下是关于这项技术的详细解析，以及为什么会有两种不同的叫法。

### 1. 原理：它们是如何工作的？

在标准的神经网络训练过程中，包含前向传播（Forward Pass）和反向传播（Backward Pass）：
*   **标准前向传播：** 模型会计算每一层的**激活值（Activations）**，并将所有中间激活值保存在显存中。
*   **标准反向传播：** 也就是计算**梯度（Gradients）**的过程，必须使用前向传播时保存的激活值。对于极深的模型，保存所有激活值会消耗巨大的显存，甚至导致 OOM（Out of Memory）。

**检查点技术（Checkpointing）的介入：**
为了节省显存，我们不保存所有的中间激活值，而是将其分成若干段。
1.  在前向传播时，只保存某些特定层（即“检查点”）的激活值，而**丢弃**两个检查点之间层的中间激活值。
2.  在反向传播时，当需要用到被丢弃的激活值来计算梯度时，模型会从最近的“检查点”重新进行一次局部的前向传播，**重新计算（Recompute）**出这些丢失的激活值。

通过这种方式，显存占用可以从 $O(n)$ 降低到近似 $O(\sqrt{n})$，代价是增加了约 20%-30% 的额外前向计算时间。

### 2. 为什么会有两个名字？

虽然指的是同一件事，但命名侧重的角度不同：

| 术语 | 侧重点 | 常见使用场景 |
| :--- | :--- | :--- |
| **激活检查点<br>(Activation Checkpointing)** | 侧重于**被保存的数据本身**。因为这项技术实际上是在内存中设立检查点来保存“激活值”（Activations），而不是保存梯度。 | 常见于 Megatron-LM、DeepSpeed 等分布式训练框架的文档中。 |
| **梯度检查点<br>(Gradient Checkpointing)** | 侧重于**该技术的最终目的**。因为我们丢弃并重新计算激活值的唯一目的，就是为了在反向传播中能够顺利计算出“梯度”（Gradients）。 | 常见于 PyTorch 官方 API（如 `torch.utils.checkpoint`）以及 Hugging Face 的文档中。 |

### 3. 代码示例 (PyTorch)

在 PyTorch 中，无论你叫它什么，调用的都是同一个 API：

```python
import torch
import torch.nn as nn
from torch.utils.checkpoint import checkpoint

class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(10, 10)
        self.layer2 = nn.Linear(10, 10)

    def forward(self, x):
        # 正常的计算，会保留激活值
        x = self.layer1(x)
      
        # 使用检查点技术：在前向时不保存 layer2 的完整中间激活值
        # 在反向传播计算梯度时，会重新执行 self.layer2(x)
        x = checkpoint(self.layer2, x, use_reentrant=False)
        return x
```

<details>
<summary><b>💡 额外补充：不要与“模型检查点 (Model Checkpointing)”混淆</b></summary>
<br>
虽然“梯度检查点”和“激活检查点”是同义词，但千万不要将它们与<b>模型检查点（Model Checkpoint）</b>混淆：

*   **Model Checkpointing（模型检查点/断点保存）：** 指的是在训练过程中，定期将模型的**权重（Weights）**和优化器状态保存到硬盘（如 `.pth`, `.safetensors`, `.ckpt` 文件）中。目的是为了防止训练中断，或者为了后续的推理（Inference）使用。这与节省显存的技术完全无关。

</details>
