### 一、 实验设计：如何追踪一个 Bad Case？

既然我们有**确定性训练（Deterministic Training）**的条件，且**数据和权重全过程可访问**，这为我们提供了物理学级别的观测条件。我们需要在“数据维度”和“模型表征维度”同时做实验。

<details>
<summary><b>展开查看：关键实验方法 (Data & Representation Tracing)</b></summary>

1. **数据归因实验 (Data Influence Tracing)**
   使用**影响函数 (Influence Functions, IF)** 或 **Datamodels**。计算 bad case 的损失梯度与训练集样本梯度的内积：
   $$ \mathcal{I}(z_{train}, z_{test}) = - \nabla_{\theta} \mathcal{L}(z_{test})^T H_{\theta}^{-1} \nabla_{\theta} \mathcal{L}(z_{train}) $$
   *实验操作：* 分别在 Base Model 和 SFT/RL 后的模型上计算该 bad case 的高影响训练样本，对比两个阶段高影响样本的分布。

2. **表征探针与因果干预 (Probing & Causal Intervention)**
   使用 **Logit Lens** 和 **Activation Patching (Causal Tracing)**。
   *实验操作：* 给定一个导致 bad case 的 prompt，记录模型在所有层、所有 token 上的残差流状态 $h_l^{(i)}$。
   然后，将 Base Model 的激活值 Patch 到 SFT 模型对应的层级中，观察 bad case 是否消失；反之亦然。

3. **权重轨迹分析 (Weight Trajectory Analysis)**
   既然训练具有确定性，我们可以计算 SFT/RL 阶段权重的变化量 $\Delta W = W_{SFT} - W_{base}$。
   *实验操作：* 对 $\Delta W$ 进行奇异值分解 (SVD)，并将其投影到导致 bad case 的输入特征方向上，观察后训练是否“摧毁”了某些预训练的正交特征。
</details>

### 二、 从现象到结论：归因矩阵

基于上述实验，我们可以观测到不同的现象，从而进行精准归因。

#### 1. 现象 A：预训练特征缺失 (The "Missing Concept" Deficit)
*   **现象：** 在针对该 bad case 相关的知识点时，Base Model 的 Logit Lens 在深层依然无法将目标 token 提升到词表 top-k。使用线性探针（Linear Probe）在 Base Model 的残差流中**无法**分离出正确的概念（Accuracy 接近随机）。Influence Function 显示，预训练数据中相关实体共现率极低。
*   **结论：这是纯粹的预训练问题（能力未建立）。** 这种情况下，无论后训练怎么做，模型只能通过 SFT 阶段的死记硬背（Memorization）来生成答案，泛化极差，极易产生幻觉。这也解释了早期 LLM RL 失败的原因：**RL 无法优化一个概率初始为 0 的动作空间。**

#### 2. 现象 B：后训练表征坍缩 (Post-Training Representation Collapse / Mode Collapse)
*   **现象：** Base Model 通过 few-shot prompting 完全可以输出正确结果。但是！在经过 SFT/RLHF 后，模型在这个 bad case 上表现出过度拒绝（Over-refusal）、格式死板或逻辑崩塌。进一步检查 $\Delta W$ 发现，SFT 阶段的更新在一个极低维的子空间内（例如方向 $v_{format}$），且这个更新方向极大地抑制了原来提取逻辑的注意力头。
*   **结论：这是纯粹的后训练问题（对齐税 Alignment Tax）。** 模型为了迎合 Reward Model 的偏好（例如详细解释、特定的 tone），破坏了原有的逻辑链。

#### 3. 现象 C：跨阶段特征纠缠 (Cross-Stage Feature Entanglement) —— *你提到的“相互作用引入新效应”*
*   **现象：** 预训练数据中存在概念 X 和概念 Y，但两者无强关联。SFT 阶段引入了特定的格式或语气 Z。在 bad case 中，模型一旦使用语气 Z 谈论 X，就会不可避免地幻觉出 Y。Activation patching 显示，SFT 强化的格式特征向量 $v_Z$ 在残差流中与预训练建立的 $v_X$ 产生了意想不到的非正交内积投影。
*   **结论：这是多阶段相互作用导致的 bad case。** 预训练留下了不完美的特征流形（X 和 Y 在几何上太近），而 SFT 提供了一个“错误推力”，导致激活轨迹滑入了错误的盆地（Basin）。

### 三、 机制刻画：超越 Induction Heads 的复杂机制

正如你所说，简单的关联回忆（Associative Recall）或 Induction Heads 无法解释 RL 阶段复杂的 bad case。我们需要更高级的几何与动力学机制来刻画：

#### 机制 1：特征压制与叠加干扰 (Feature Suppression & Superposition Interference)
预训练模型为了将超大规模的世界知识压缩进有限的维度，大量使用了**特征叠加 (Superposition)**。即 $N$ 个特征被压缩在 $d$ 维空间中 ($N \gg d$)。
在后训练阶段，为了强化某些“安全”或“推理格式”特征，RL 的梯度会大规模更新某些方向。
$$ h_{new} = h_{base} + \alpha \Delta W x $$
如果 $\Delta W$ 的主成分方向与某些预训练中处于 Superposition 的罕见知识特征不小心**反向对齐**（或产生严重的非正交干扰），就会导致该知识被永久“压制”。这就解释了为什么分阶段训练会引入之前没有的 bad case：**因为后训练打破了预训练脆弱的特征正交平衡。**

#### 机制 2：注意力头被“劫持” (Attention Head Hijacking)
在 Mechanistic Interpretability 中，我们发现 LLM 中存在特定的电路（Circuits）。预训练中，某些深层的 Attention Heads（比如 Head L20-H5）负责“逻辑校验（Logic Checking）”。
但是在复杂的 RL 后训练中（尤其是强化特定输出模式时），优化器发现**修改 Head L20-H5 的 value matrix $W_V$ 以输出“格式 Token”是降低 Loss 最快的捷径**。
结果，这个“逻辑头”被劫持成了“格式头”。Bad case 表现为：模型格式极度完美，但逻辑出现低级错误，因为负责逻辑的电路在 RL 中被覆写了。

<details>
<summary><b>展开查看：机制 3 - 隐空间流形的局部断裂</b></summary>
预训练构建了一个连续的概率流形 $\mathcal{M}$。好的推理（如 R1）需要在 $\mathcal{M}$ 上进行漫长的游走（产生隐式思维链）。
如果在预训练中，Math 和 Coding 的知识流形之间存在“断裂带（Low Density Region）”，而后续 SFT 的强制拼接让模型在这个断裂带中游走，模型的隐状态就会变成“乱码（Gibberish）”，最终解码出幻觉。这再次证明：<b>后训练算法越复杂，越容易把模型推入未知的断裂带；底模越强（流形越连续），后训练就越简单。</b>
</details>

### 四、 总结与研究建议

你的直觉——**“所有的 bad cases，（归根结底）都是因为预训练得不好”**——在表征工程（Representation Engineering）的视角下是非常深刻的。

我们可以用一句话总结：
> **预训练决定了模型特征空间（Feature Space）的基底和基底之间的正交性；后训练只是在这个空间中求解一个路由策略（Routing Policy）。**

*   **如果预训练好：** 特征丰富且解耦良好，后训练只需要很小的权重变动（Small Norm Update）就能组装出强大的能力（LIMA / R1 的现象）。
*   **如果预训练不好：** 存在特征缺失或严重的非正交耦合。此时后训练必须通过大幅修改权重（Large Norm Update）去强行拟合，这必然会导致“劫持电路”、“特征压制”，从而涌现出各阶段单独存在时都没有的 Bad Cases。

**针对你们的研究，我建议的操作路径：**
1. 挑出几个典型的“跨阶段相互作用”导致的 Bad Case。
2. 使用 `Logit Lens` 和 `Linear Probes` 确定出错的具体层级。
3. 对 SFT/RL 增量矩阵 $\Delta W$ 做低秩分解，将其作用于 Base Model 的残差流。
4. 观察这个 $\Delta W$ 投影是否“破坏（破坏了正交性）”或“掩盖（绝对值压制）”了 Base Model 原本正确的特征向量。如果能用数学（内积或范数变化）证明这一点，将是一个非常有影响力的工作。

### 🤖 Assistant

你的洞察非常精准。试图去解释 Attention 权重、甚至去寻找具体的 Circuit，在现代大模型（特别是带有 Sparse Attention、GQA、MoE 的架构）中确实已经走到了死胡同。由于**特征叠加（Superposition）**的存在，单个神经元或单个 Head 根本不具备单一可解释性（Monosemanticity）。

既然抛弃了权重层面，我们就必须把 LLM 看作是一个**动态系统（Dynamical System）**，将研究的焦点完全转移到**残差流（Residual Stream, 也就是 Hidden States $h_l \in \mathbb{R}^d$）**上。

要想在 Representation 中找到“特定性质”的**方向（Direction）**和**大小（Magnitude/Activation）**，目前学术界有两条被证明最行之有效的路径：**对比表征工程（Contrastive RepE）**和**稀疏自编码器（Sparse Autoencoders, SAEs）**。

---

### 方法一：对比主成分分析（Contrastive PCA / RepE）—— 寻找宏观性质方向

这是目前最实用、计算代价最低的方法（例如邹昊团队的 Representation Engineering）。我们不需要管 Attention 怎么算的，只看结果表征。

**1. 寻找方向（Direction $v$）：**
要找到某个性质（比如“遵循指令”、“产生幻觉”、“某个数学公理”）的方向，我们可以构造一对 Contrastive Dataset：
*   $X_{pos}$：触发该性质的 Prompt（例如正确的推理链）。
*   $X_{neg}$：缺乏该性质或对立的 Prompt（例如胡说八道的推理链）。

将这两组数据输入模型，提取某一层 $l$ 最后一个 token 的残差流表征集合 $H_{pos}$ 和 $H_{neg}$。
计算均值差（Mean Difference）或做主成分分析（PCA）：
$$ v_{property} = \frac{1}{N} \sum_{i=1}^N h_{pos}^{(i)} - \frac{1}{N} \sum_{i=1}^N h_{neg}^{(i)} $$
或者取 $H_{pos} - H_{neg}$ 的第一主成分。这个单位向量 $v_{property} \in \mathbb{R}^d$ 就是该性质在潜空间中的**方向**。

**2. 测量大小（Magnitude $s$）：**
对于任何一个引发 Bad Case 的输入，提取其表征 $h_{bad}$，它在该性质上的激活大小即为投影（内积）：
$$ s = h_{bad} \cdot v_{property} $$

<details>
<summary><b>展开查看：如何用该方法诊断 Pre-training vs Post-training？</b></summary>

假设引发 Bad Case 的原因是模型在某个步骤突然“遗忘”了某个前置条件（导致逻辑断裂）。
1. 我们用上述方法找出代表“该前置条件”的方向 $v_{concept}$。
2. 绘制 Base Model 和 RL Model 在处理该 Bad Case 时，每一层的投影大小 $s_{base}^{(l)}$ 和 $s_{RL}^{(l)}$。
*   **诊断 1（预训练背锅）：** 如果 $s_{base}^{(l)}$ 在所有深层都接近 0，说明底模根本没有提取出这个特征。
*   **诊断 2（后训练背锅）：** 如果 $s_{base}^{(l)}$ 很大，但 RL Model 在某一层（比如负责输出特定句式的层）之后，$s_{RL}^{(l)}$ 突然暴跌。这就证明 RL 引入的权重更新 $\Delta W$ 产生了一个与 $v_{concept}$ 负相关的扰动向量，在残差流中**抵消（Cancel out）**了原本正确的特征。
</details>

### 方法二：稀疏自编码器（Sparse Autoencoders, SAEs）—— 破解特征叠加（Anthropic路线）

如果你觉得 PCA 找出的方向依然是宏观概念的混杂（Entanglement），那么目前**唯一能在微观上提取干净方向**的技术就是 SAE（字典学习）。

既然 LLM 为了省空间，把几千万个特征压缩到了几千维的残差流 $h$ 中，SAE 的逻辑就是训练一个极宽的浅层网络，把 $h$ 解压缩回高维的、稀疏的、**单语义**的特征空间。

$$ h \approx \sum_{i=1}^{M} f_i d_i $$

*   $d_i \in \mathbb{R}^d$ 是字典里的第 $i$ 个**方向（Feature Direction）**。
*   $f_i = \text{ReLU}(W_{enc} h + b)_i$ 是该方向上的**大小（Feature Activation）**。

**如何应用在你的 Bad Case 归因研究中？**

1.  用你的 Base Model 和 SFT Model 的残差流分别训练（或复用开源的）SAE。
2.  输入 Bad Case，观察 SAE 的激活模式。此时你不再是看 $h$ 这个密集的、不可解释的向量，而是看到几十个具体的 $f_i > 0$ 的可解释特征。
3.  **发现相互作用的幽灵：** 你可能会在引发 Bad Case 的那一刻，发现 SFT Model 中激活了一个“特定的语气特征 $d_{tone}$”，而**正是这个特征的激活，抑制了“逻辑推理特征 $d_{logic}$”的激活**。而在 Base Model 中，由于没有 $d_{tone}$ 的强制介入，$d_{logic}$ 得以平稳存活到最后一层。

### 实验设计：Causal Tracing（如何证明你找的方向是对的？）

找到了方向和大小，你必须证明它确实是导致 Bad Case 的因果机制，这就需要做**激活修补（Activation Patching / Intervention）**。

不要去改 Attention 权重，直接改残差流：
$$ h_{modified}^{(l)} = h_{original}^{(l)} + \alpha \cdot v_{property} $$
或者直接将 Base Model 中含有正确信息的表征，强行替换掉 RL Model 中被破坏的表征（在特定的 Layer 和 Token 上）。

如果替换某个特定方向 $v$ 后，模型的 Bad Case 瞬间被修复，或者故意反向注入 $-v$ 让原本表现良好的 Base Model 产生一模一样的 Bad Case，你就完成了一个**从表征层面完美闭环的归因**。

### 总结

现代的可解释性已经变成了**潜空间的几何学**：
*   **预训练**是在雕刻这个隐空间的高维流形（Manifold），决定了哪些方向 $v$ 是存在的。
*   **后训练（RL）**是在引入新的引导向量（Steering Vectors），改变模型在流形上的游走轨迹。
*   **跨阶段相互作用的本质**，是 RL 引入的引导向量在几何上与预训练保留下来的脆弱逻辑向量不垂直，产生了意外的**投影干扰（Interference）**。通过对比 RepE 投影或 SAE 激活，你能把这种极其抽象的“相互作用”用具体的内积 $s = h \cdot v$ 算出来。
