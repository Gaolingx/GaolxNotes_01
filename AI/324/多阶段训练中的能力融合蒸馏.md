# 多阶段训练中的能力融合蒸馏

### 🧑‍💻 User

llm多阶段训练（SFT+RL（如 Reasoning RL、General RL））是按不同目标顺序优化的。
这种优化容易出现一个问题：后面阶段学新东西时，会把前面阶段已经学好的能力削弱，形成能力的退化。

所以很多sota的pipeline在最后加了一个 On-Policy Cross-Stage Distillation 阶段，能否说说核心目标，数据来源和算法实现？

---

### 🤖 Assistant

在LLM的多阶段训练（如 Pre-training $\rightarrow$ SFT $\rightarrow$ Reasoning RL $\rightarrow$ General RL/Alignment）中，你提到的现象在学术界被称为**“对齐税”（Alignment Tax）**或**“灾难性遗忘”（Catastrophic Forgetting）**。当模型在后期阶段（如专门针对数学和代码的 Reasoning RL）过度优化特定目标时，其早期的通用对话能力、格式遵循能力或创造性写作能力往往会退化。

为了解决这个问题，SOTA 的 pipeline（如 Llama 3、Nemotron 等的训练思路中）常在最后引入 **On-Policy Cross-Stage Distillation（同策略跨阶段蒸馏）**。

以下是该阶段的核心目标、数据来源和算法实现的详细解析：

### 一、 核心目标 (Core Objectives)

引入这一最后阶段的根本目的是**“既要又要”**：保留 RL 阶段获得的极致推理能力，同时恢复/巩固 SFT 阶段的通用能力。

1.  **缓解“对齐税”与灾难性遗忘：** 将早期阶段（SFT 或初期 General RL）的概率分布知识“拉回”到当前模型中，修复因过度强化特定任务（如代码、数学）而导致的通用语言能力受损。
2.  **平滑概率分布 (Smoothing Distribution)：** RL（尤其是 PPO 算法）容易导致模型输出的概率分布变得极其尖锐（只输出特定高优 token）。通过蒸馏，可以恢复模型输出的多样性和鲁棒性。
3.  **统一能力范式：** 将多个特定领域的专家能力（如专精数学的 RL Checkpoint 和专精代码的 RL Checkpoint）与通用基础能力融合到一个统一的最终模型（Student）中。

---

<details>
<summary><b>二、 数据来源 (Data Sources) —— 为什么强调 On-Policy？</b></summary>
<br>

传统蒸馏通常使用固定的离线数据集（Off-Policy），但这会导致**分布偏移（Distribution Shift）**：教师模型的评分是在静态数据上做出的，而学生模型在实际推理时可能会生成完全不同的轨迹。

**On-Policy (同策略)** 意味着数据是由**当前正在训练的学生模型**实时生成的。

1.  **Prompt 来源 (混合池)：**
    *   构建一个混合 Prompt 池，包含 SFT 阶段的通用任务（写作、问答、角色扮演）和 RL 阶段的复杂任务（数学、代码、逻辑推理）。
2.  **Response 生成 (On-Policy Rollout)：**
    *   使用**当前正在训练的模型 (Student Policy)** 对这些 Prompt 进行推理，生成 Response（轨迹 $y$）。
3.  **Teacher 标注 (Cross-Stage Labeling)：**
    *   **对于通用 Prompt：** 将生成的 Response 送入 **SFT 模型（Teacher）**，计算每个 token 的 Logits / 概率分布。
    *   **对于推理 Prompt：** 将生成的 Response 送入 **Reasoning RL 模型（Teacher）**，计算 Logits。
    *(有时也采用直接获取 Teacher 生成的 Hard Label 作为参考，但 Soft Label / Logits 蒸馏效果更好)*。

</details>

<details>
<summary><b>三、 算法实现 (Algorithm Implementation)</b></summary>
<br>

在算法实现上，这本质上是一个**以 KL 散度（KL Divergence）为核心的多教师约束优化过程**。

#### 1. 模型设定
*   **Student Model ($\pi_{\theta}$)**：当前正在训练/微调的最终模型。
*   **Teacher Model 1 ($\pi_{SFT}$)**：跨阶段的早期模型（负责提供通用能力分布）。
*   **Teacher Model 2 ($\pi_{RL}$)**：跨阶段的特定能力模型（负责提供推理能力分布）。

#### 2. 损失函数设计 (Loss Function)
对于给定的 Prompt $x$，当前学生模型生成轨迹 $y = (y_1, y_2, ..., y_T)$。算法通过最小化学生分布和教师分布之间的 KL 散度来更新参数。

通常使用 Forward KL (即以 Teacher 的分布指导 Student) 或 Reverse KL。在 On-Policy 场景下，我们通常优化以下目标：

$$ \mathcal{L}_{distill}(\theta) = \sum_{x \in \mathcal{D}} \mathbb{E}_{y \sim \pi_{\theta}(\cdot|x)} \left[ \sum_{t=1}^{T} D_{KL} \left( \pi_{teacher}(\cdot|x, y_{<t}) \parallel \pi_{\theta}(\cdot|x, y_{<t}) \right) \right] $$

*   $\pi_{\theta}$ 是正在生成轨迹的 On-Policy 学生模型。
*   $\pi_{teacher}$ 根据 $x$ 的领域动态切换。如果 $x$ 是通用对话，则 $\pi_{teacher} = \pi_{SFT}$；如果 $x$ 是数学题，则 $\pi_{teacher} = \pi_{RL}$。

#### 3. 算法执行流程 (伪代码思路)
```python
for batch_prompts in dataloader:
    # 1. On-Policy Generation (同策略生成)
    responses = student_model.generate(batch_prompts) 
  
    # 2. Cross-Stage Teacher Scoring (跨阶段教师评分)
    teacher_logits = []
    for prompt, response in zip(batch_prompts, responses):
        if is_general_domain(prompt):
            # 用早期的 SFT 模型做 Teacher
            logits = sft_model.forward(prompt, response)
        else:
            # 用后期的 RL 模型做 Teacher
            logits = reasoning_rl_model.forward(prompt, response)
        teacher_logits.append(logits)
      
    # 3. Student Scoring (学生评分)
    student_logits = student_model.forward(batch_prompts, responses)
  
    # 4. Calculate Loss & Update (计算KL散度并更新参数)
    loss = compute_kl_divergence(teacher_logits, student_logits)
    loss.backward()
    optimizer.step()
```

#### 4. 变体：带有动态权重的目标融合
在某些实现中，为了不完全放弃继续强化学习的机会，会将 RL 损失（如 PPO 的 Surrogate Loss 或 DPO Loss）与跨阶段蒸馏损失（KL Loss）相加，作为最终优化目标：

$$ \mathcal{L}_{total} = \mathcal{L}_{RL}(\theta) + \beta \cdot \mathcal{L}_{distill}(\theta) $$

其中 $\beta$ 是一个动态调节系数。当模型在通用任务上表现下降时，增大 $\beta$ 强制拉回 SFT 分布。

</details>

### 总结
On-Policy Cross-Stage Distillation 的核心哲学是**“温故而知新”**。通过让模型“自己做题”（On-Policy），然后请“不同阶段的过去的自己”（Cross-Stage Teachers）来“批改和打分”（Distillation），从而完美地将 RL 阶段习得的深刻逻辑与 SFT 阶段沉淀的广博知识缝合在一起，最终交付一个全能的 SOTA 模型。

为了更透彻地理解为什么这三点组合起来如此强大，我们可以将其与传统的离线方法（Off-Policy）和普通的强化学习（RL）做一个对比剖析：

### 1. 为什么“在自己的轨迹上学习”（On-Policy）极其关键？

传统解决遗忘的方法是“把 SFT 数据拿出来再训练一遍”（即 Off-Policy SFT / Replay）。但这种方法往往效果不佳，核心原因在于 **`Exposure Bias`（暴露偏差）**。

<details>
<summary><b>💡 展开：Off-Policy 与 On-Policy 的本质区别</b></summary>
<br>

*   **Off-Policy (离线/标准SFT)：** 模型只看到人工写好的“完美标准答案”。一旦模型在实际生成中输出了一个偏离标准轨迹的 token，它就会进入一个它从未在训练中见过的状态空间，从而不知所措，导致错误累积。
*   **On-Policy (同策略蒸馏)：** 学生模型先按照自己**当前的脑回路（Policy）**去生成回答。即使它走偏了，或者用了一种比较啰嗦的方式回答，教师模型也会基于这个“偏离的轨迹”给出下一步的概率指导。
    *   *通俗比喻：* Off-Policy 就像老师只让你抄标准答案；On-Policy 就像你在自己做题，老师在你写错的步骤旁边，告诉你“既然你已经写到这一步了，接下来最合理的补救/继续方式是什么”。这种方式教会了模型如何**在自身当前的分布下**保持通用能力。

</details>

### 2. 为什么“密集、逐步的纠错信号”比 RL 奖励更好？

强化学习（如 PPO）容易导致能力退化，是因为它的信号太**稀疏（Sparse）**且**片面**。
RL 通常只在生成结束后给一个标量分数（Reward），模型为了拿高分，会疯狂走捷径（Reward Hacking），导致输出分布极其尖锐（即只输出能拿高分的特定词汇，丧失了语言的多样性和通用能力）。

*   **教师提供的密集信号（Dark Knowledge）：** 跨阶段蒸馏提供的是教师模型在**每一个 Token 上的完整概率分布（Logits）**。
*   **平滑性保障：** 比如在生成“苹果是__”时，RL 可能只给“红色的”打高分。但 SFT 教师模型会告诉学生：“红色的”概率是 $0.6$，“一种水果”是 $0.3$，“科技公司”是 $0.09$。这种包含次优选项概率的信号（即 **Dark Knowledge**），极大地保护了模型输出的丰富度和语境适应能力，防止了分布塌陷。

### 3. 为什么是“重新对齐之前的各教师模型”（Cross-Stage）？

在多阶段训练中，模型就像一个不断转学的学生：
1.  **SFT 阶段：** 学会了礼貌、格式化输出、写诗、翻译（通才）。
2.  **Reasoning RL 阶段：** 被送进奥数班和信奥班，变成了只懂死磕逻辑的理科狂人（专才），开始忘记怎么写诗和日常聊天。

如果在最后阶段不引入**跨阶段（Cross-Stage）**的教师，模型就会永久丧失部分 SFT 能力。
通过 On-Policy Cross-Stage Distillation，系统充当了一个**“能力调度中心”**：
*   遇到数学题轨迹 $\rightarrow$ 呼叫 RL Teacher，巩固逻辑。
*   遇到日常问答轨迹 $\rightarrow$ 呼叫 SFT Teacher，唤醒通用认知。

**总结来说：**
On-Policy Distillation 创造了一个完美的闭环——模型自己探索（On-Policy），过往的“巅峰状态自己”（Cross-stage Teachers）手把手纠正每一个细节（Dense Logits），从而在不损失新学到的硬核推理能力的前提下，完美“缝合”并找回了曾经的通用能力。

### 🧑‍💻 User

在持续学习中，模型需要在获得新知识的同时尽量保持已有能力。传统 SFT 和一般 RL 都难以单独满足这一目标。

off-policy训练（如SFT） 的问题在于，它直接拟合新任务标注分布，容易把模型参数过度拉向新数据，从而破坏原有能力并导致灾难性遗忘。尤其是在知识注入场景下，SFT 往往缺乏对旧能力的显式约束，因此容易出现“学会新知识、忘掉旧能力”的现象。

一般 RL 虽然由于采用 on-policy 采样，通常比 SFT 更不容易遗忘，但它主要依赖稀疏奖励来优化行为，擅长“让模型做得更对”，却不擅长细粒度、密集地传递新知识。因此，RL 往往训练成本更高、样本效率更低，也较难高效完成知识注入。

相比之下 OPD 为什么非常有望成为持续学习的新范式？

### 🤖 Assistant

你的分析非常深刻！你精准地指出了 SFT（过度拟合导致参数剧变）和一般 RL（稀疏奖励导致知识注入效率极低）在持续学习（Continual Learning, CL）中的核心痛点。

基于你的逻辑，**On-Policy Distillation (OPD)** 之所以极具潜力成为大模型持续学习的新范式，是因为它完美地在这个看似矛盾的“不可能三角”（注入新知识、保持旧能力、高样本效率）中找到了平衡点。

具体来说，OPD 成为持续学习新范式的核心优势可以归结为以下三个维度：

### 1. 细粒度的密集信号：突破 RL 的低效瓶颈

一般 RL（如 PPO）通过标量奖励（Scalar Reward）来反馈，这就像只在考试结束后给出一个总分，模型很难弄清楚到底是哪一步写得好或不好，尤其在学习具体的“事实性新知识”时，犹如盲人摸象。

*   **OPD 的解法：** OPD 通过教师模型提供**Token 级别的 Logits / 概率分布**（即 $\pi_{teacher}$）。这是一种密集（Dense）的指导信号。
*   **CL 视角的优势：** 当模型需要注入新领域知识时，新领域的专家教师（或经过精心构造的高质量参考）可以在模型生成的每一个词上提供精确的梯度指导。这种 Token 级别的交叉熵/KL 散度优化，其**样本效率和知识注入能力几乎等同于 SFT**，完美弥补了 RL 的短板。

### 2. 显式的旧能力约束（Functional Regularization）：突破 SFT 的遗忘困境

SFT 是毫无顾忌地将模型参数拉向新数据的分布，没有任何机制告诉模型“哪些旧参数/旧分布是不能动的”。

*   **OPD 的解法：** 在 OPD 框架下，可以极度自然地引入**“过去的自己”（历史 Checkpoint）作为守护教师**。在训练新知识的同时，混合一部分旧任务的 Prompt。对于这部分旧 Prompt，强制要求当前学生模型的输出分布 $\pi_{\theta}$ 与历史教师模型的分布 $\pi_{old}$ 之间的 KL 散度最小化（$D_{KL}(\pi_{old} \parallel \pi_{\theta})$）。
*   **CL 视角的优势：** 这在数学上等价于一种**函数空间正则化（Functional Regularization）**。它显式地在损失函数中加上了一道锁，告诉优化器：“你可以尽情改变参数去拟合新知识，但前提是，在旧任务的 Prompt 空间里，你的输出概率分布必须和以前保持一致。”这就从根本上阻断了灾难性遗忘。

### 3. 基于当前状态流形（Manifold）的平滑演进

Off-policy SFT 最危险的地方在于，它强迫模型去模仿别人的轨迹（人类标注者的语气的遣词造句），这常常导致模型被拉拽到它自身当前能力流形（Manifold）之外的参数空间，从而造成“能力撕裂”。

*   **OPD 的解法：** 因为是 On-Policy，模型始终在**“自己生成的轨迹”**上计算 Loss。
*   **CL 视角的优势：** 在持续学习中，这意味着模型是在**以自己现有的认知结构和表达习惯为基础，去吸收新知识**。即使引入了新知识的教师信号，这个信号也是作用在模型自己产生的轨迹上的（即“在模型的舒适区内进行微调”）。这种平滑的参数更新方式，极大减小了分布偏移带来的破坏力。

### 总结与展望

我们可以用一个简单的对比来总结为什么 OPD 是持续学习的理想范式：

| 训练范式 | 知识注入效率 | 旧能力保持（防遗忘） | 优化轨迹 | 在持续学习中的表现 |
| :--- | :--- | :--- | :--- | :--- |
| **Off-Policy SFT** | **高** (密集标签) | **差** (无约束，易覆盖参数) | 强制模仿外部轨迹 | 容易学得快，忘得快 |
| **一般 RL** | **低** (稀疏奖励) | **中** (On-policy 减缓突变) | 探索自身轨迹 | 成本过高，难塞入新知识 |
| **OPD (新范式)** | **高** (密集 Logits) | **极佳** (多教师显式 KL 约束)| 探索自身轨迹 | **高效率注入，且平滑不遗忘** |

在未来的 LLM 生命周期管理中，模型不可能每学一个新领域（如法律、医疗）就从头 Pre-training 或大规模 SFT。**OPD 提供了一种类似于人类“温故知新”的学习机制**：用当前的大脑去思考问题（On-Policy），请新领域的专家纠正新知识（Dense Signal），同时请过去的自己复习旧知识（KL Constraint），这无疑是目前最具前景的持续学习路线。
