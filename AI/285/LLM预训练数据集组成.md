# LLM预训练数据集组成

### 🧑‍💻 User

llm预训练到底使用什么数据集？finewebedu ？HuggingFaceTB/cosmopedia？

---

### 🤖 Assistant

LLM（大语言模型）的预训练数据**绝不是单一的数据集**，而是一个精心按比例调配的**“数据混合体”（Data Mixture）**。

你提到的 `FineWeb-Edu` 和 `HuggingFaceTB/cosmopedia` 都是目前开源社区非常前沿、高质量的预训练数据集，但它们代表的是**当前预训练数据发展的新趋势（高质量过滤与合成数据）**，而不是LLM预训练的全部。

下面为你详细拆解LLM预训练到底在用什么数据，以及你提到的这两个数据集处于什么位置。

### 1. 你提到的两个数据集代表了什么？

这两个数据集都来自 Hugging Face 团队，代表了目前小规模/中规模开源模型预训练的最优解之一：

*   **`FineWeb-Edu`（极致清洗的高质量真实数据）：**
    它是从庞大的 `FineWeb`（一个海量的网页抓取数据集）中进一步提纯出来的。Hugging Face 使用大模型（如 Llama-3）训练了一个“教育价值分类器”，给网页打分，只保留那些具有极高教育意义、知识密度高的网页。**作用：** 用它训练的模型（如 Smaug 系列、部分近期的微型模型）在各项基准测试中表现极好，证明了“数据质量远比数量重要”。
*   **`HuggingFaceTB/cosmopedia`（完全的合成数据）：**
    这是一个由 Mixtral-8x7B 模型生成的**千万级全合成数据集**，包含了生成的教科书、博客文章、故事和代码。**作用：** 它是用来探索“纯合成数据预训练”潜力的。Hugging Face 用它训练了 `Cosmo-1B` 模型，证明了仅靠合成数据就能让小模型具备极强的常识推理和连贯生成能力（类似微软 Phi 系列的路线）。

---

### 2. 顶级大厂（如 OpenAI, Google, Meta）的“标准预训练配方”

无论是 GPT-4、Gemini 还是 Llama 3，它们的预训练数据配方虽然保密，但基本结构是高度一致的。通常包含以下几个大类（按 token 占比从大到小）：

1.  **网页抓取数据 (Web Crawl) - 占比最大 (约 60%-80%)**
    *   **来源：** 主要是 Common Crawl（CC）。
    *   **处理：** 原始 CC 数据充满垃圾信息。大厂会花费极大精力进行去重（MinHash）、去毒、语言分类、质量过滤。`FineWeb` 就是开源界目前对 CC 清洗得最好的版本之一。
2.  **代码数据 (Code) - 逻辑推理的基石 (约 10%-15%)**
    *   **来源：** GitHub、StackOverflow 等开源代码库。
    *   **作用：** 惊人的是，加入大量代码数据不仅能让模型写代码，还能**显著提升模型的逻辑推理能力和结构化思维**。
3.  **高质量文本与图书 (Books & High-quality text) - 深度阅读理解 (约 10%)**
    *   **来源：** 版权书籍、Project Gutenberg、Smashwords 等。
    *   **作用：** 提供长上下文的连贯性、复杂的句法结构和深度的世界知识。
4.  **百科与学术论文 (Wikipedia & ArXiv) - 事实知识与专业领域 (约 5%)**
    *   **来源：** 维基百科（多语种）、ArXiv（包含 LaTeX 公式）、PubMed（医学）、USPTO（专利）。
    *   **作用：** 注入硬核的事实、科学知识和数学能力。

---

### 3. LLM 预训练数据的发展趋势

目前的预训练已经从**“越大越好 (Scale is all you need)”**转向了**“质量至上 (Quality is all you need)”**。

*   **过去（2020-2022）：** 直接拿清洗过的 Common Crawl 暴力训练（如 C4, The Pile）。数据量大，但包含很多废话。
*   **现在（2023-至今）：**
    1.  **教科书级别的数据（Textbook Quality）：** 微软在训练 Phi 模型时提出了这个概念，即只用极高质量的、像教科书一样严谨的数据。`FineWeb-Edu` 就是这一理念的开源实现。
    2.  **合成数据（Synthetic Data）：** 真实世界的高质量数据快被大厂用光了（所谓的 Data Wall）。因此，用强模型（如 GPT-4）生成教科书、逻辑题来训练弱模型成为了主流。`cosmopedia` 就是这方面的代表。

如果你想了解除了你提到的两个之外，开源界还有哪些“老牌”和“经典”的预训练数据集：

*   **The Pile (EleutherAI):** 早期非常经典的 800GB 高质量数据集，包含了 22 个不同的数据源（PubMed, ArXiv, GitHub 等），对早期开源模型贡献巨大。
*   **C4 (Colossal Clean Crawled Corpus):** Google 提出，用于训练 T5。是经过基础清洗的网页数据。
*   **RedPajama:** 根据 Llama 1 的论文公开的配方，社区复现的 1.2 Trillion tokens 的开源数据集，包含了维基百科、书籍、ArXiv、代码等。
*   **RefinedWeb (Falcon):** 用极度严苛的启发式规则清洗的网页数据，证明了哪怕只用网页数据，只要洗得够干净，也能训练出好模型。
*   **The Stack (BigCode):** 目前最全面、合规的开源代码预训练数据集，用于训练 StarCoder 等代码模型。

</details>

### 总结

如果你现在要**从头预训练 (Pre-train)** 或**持续预训练 (Continual Pre-training)** 一个模型：

*   使用 **`FineWeb-Edu`** 作为你的核心通用知识和网页数据来源是非常明智的，它是目前开源界最好的选择之一。
*   使用 **`HuggingFaceTB/cosmopedia`** 可以作为很好的补充，用来增强模型的逻辑、常识和对话连贯性。
*   但千万别忘了，你还需要加入**高质量的代码数据集**（如 `The Stack v2` 或 `StarCoderData`）以及**数学文献**，否则你的模型会缺乏逻辑推理能力。

---

### 🤖 Assistant

这两种路线代表了目前大模型预训练中关于**“数据调度（Data Curriculum）”**的不同哲学。

直接给出结论：**强烈推荐“路线1：多阶段分层预训练（Curriculum Learning / Data Annealing）”**。这也是目前包括 Meta (Llama 3)、Mistral 以及国内主流大厂（如 DeepSeek、Qwen）在冲刺极致性能时普遍采用的策略。

下面为你深度剖析这两种路线的优劣，以及为什么业界正在向路线1靠拢。

---

### 路线 1：两阶段/多阶段预训练（兼顾广度与深度）
*核心思想：好钢用在刀刃上，让模型“先学认字，再学逻辑”。*

这种路线通常分为两个或三个阶段：
*   **Stage 1 (基础预训练)：** 使用海量、占比最大的泛领域数据（如清洗后的 Common Crawl，占据 70%-80% 的 token）。目的是让模型掌握语言的统计规律、基础语法和极其广泛的世界知识。
*   **Stage 2 (退火/高质量持续预训练 - Annealing)：** 在训练的最后阶段（通常是最后 5%~10% 的 token），大幅提高高质量数据（如 `FineWeb-Edu`、数学、代码、`cosmopedia` 等合成数据）的比例，甚至纯用高质量数据。

**推荐理由（优点）：**
1.  **极大地提升了高质量数据的利用率：** 高质量的合成数据和教科书数据非常稀缺且昂贵。如果在模型刚开始训练、Loss 值很高（还在“学认字”）的时候喂给它，纯属浪费。等模型具备了基础认知，再喂给它逻辑严密的数据，它能更深层次地吸收其中的推理能力。
2.  **避免知识污染：** 合成数据虽然逻辑好，但词汇多样性和世界知识（广度）不如自然语料。先用自然语料打底，再用合成数据拔高，可以完美互补。
3.  **对齐 SFT（指令微调）的平滑过渡：** 第二阶段实际上起到了 Pre-training 和 SFT 之间的桥梁作用。

**缺点：**
*   工程复杂度稍高。你需要在不同阶段切换数据加载器，并且要非常小心地处理**学习率（Learning Rate）**。

<details>
<summary><b>🛠️ 展开查看：两阶段训练中的学习率陷阱 (Learning Rate Schedule)</b></summary>

在多阶段训练中，最大的坑在于学习率的控制。通常预训练使用余弦退火（Cosine Decay）策略：
$$ lr_t = \eta_{min} + \frac{1}{2}(\eta_{max} - \eta_{min})\left(1 + \cos\left(\frac{T_{cur}}{T_{max}}\pi\right)\right) $$

*   **陷阱：** 如果你在第一阶段结束时，学习率已经衰减到了接近 $0$，此时直接无缝切换到高质量的 Stage 2，模型几乎学不到新东西，因为步长太小了。
*   **正确做法：** 
    1.  在进入 Stage 2 时，进行一次轻微的**学习率预热（Warmup）**或重置，然后再快速衰减。
    2.  或者像 Llama 3 一样，在标准预训练结束后，不急着结束，保持一个较小的恒定学习率，专门跑高质量数据进行“数据退火（Data Annealing）”。

</details>

---

### 路线 2：一镜到底的多领域合成/自然语料混合
*核心思想：寻找“黄金比例”，大道至简。*

这是早期（如 Llama 1/2 时代）比较传统的做法。在训练前，算法工程师通过大量的小规模实验（Scaling Laws）算出最优的数据配比（例如：60% 网页 + 15% 代码 + 10% 书籍 + ...），混合打乱后，跑完几 Trillion 的 token 结束。

**优点：**
1.  **Pipeline 极其精简：** 训练脚本跑起来之后基本不用管，不需要中途停下来换数据、调学习率。
2.  **防止遗忘：** 领域知识均匀分布在整个训练周期中，理论上不存在“学了后面忘了前面”（灾难性遗忘）的问题。

**不推荐的原因（缺点）：**
1.  **“黄金比例”极其难找：** 在不同参数规模下，模型对各类数据的需求比例是动态变化的。一镜到底意味着你必须在训练前就押注一个固定的比例。如果比例错了（比如代码加少了），你只能眼睁睁看着几百万美元的算力打水漂，或者只能在后期强行做 Continual Pre-training 补救。
2.  **合成数据的早期浪费：** 如前文所述，在模型初始阶段投入高逻辑密度的合成数据收益极低。

---

### 总结与建议

如果你或者你的团队准备训练一个模型：

1.  **如果你只有极少量的算力，想用最小代价验证想法：**
    可以选择**路线2**，但数据池可以极其精简（比如直接用别人调配好的高质量开源 Mixture），快速跑完。
2.  **如果你想冲击榜单，或者追求同等参数量下的极致性能（Best Practice）：**
    **必须采用路线1。** 
    *   **Phase 1 (0% - 90% tokens):** 泛用网页数据（如 CC 清洗版） + 基础代码 + 书籍百科。
    *   **Phase 2 (90% - 100% tokens):** 极高比例的 `FineWeb-Edu` + `HuggingFaceTB/cosmopedia` + 复杂数学证明（如 OpenWebMath） + 高级算法代码。在这个阶段，模型会发生“顿悟”（Grokking），其逻辑推理测试集（如 GSM8K, MMLU）的分数通常会有一个极其陡峭的拉升。
