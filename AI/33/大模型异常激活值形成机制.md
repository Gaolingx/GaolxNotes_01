# 大模型异常激活值形成机制

### 🧑‍💻 User

近期的研究（如 LLM.int8(), SmoothQuant, 以及关于 Attention Registers 的论文）发现，当模型参数量扩展到一定规模（通常在 6B-13B 以上）时，Transformer 会自发地演化出一种机制：在少数特定的隐藏维度（Hidden Dimensions）上产生极大的激活值（往往比平均值大上百倍）。这种异常激活值是怎么形成的，为什么微小的扰动或舍入会对模型性能产生较大影响？

---

### 🤖 Assistant

在大型语言模型（LLM）的缩放法则（Scaling Law）中，当模型参数量突破 6B-13B 的临界点时，隐藏层激活值（Hidden Activations）会出现明显的**相变（Phase Transition）现象**。即在极少数特定的隐藏维度（Channels/Dimensions）上，会自发涌现出极大的异常值（Outliers），其幅值往往是其他维度的百倍以上。

这种现象不仅是 `LLM.int8()` 和 `SmoothQuant` 等量化研究的核心痛点，也被 Vision Transformers (ViT) 和 LLM 中的 *Attention Registers* 论文从机制上进行了解释。

以下是关于这些异常激活值形成原因及其对扰动敏感的详细解析。

### 一、 异常激活值（Outlier Features）是怎么形成的？

这些异常维度并非模型的“缺陷”，而是 Transformer 在大规模参数下为了优化信息流动而**自发演化出的一种全局路由（Global Routing）和信息存储机制**。

#### 1. Attention 机制的“注意力池（Attention Sinks / Registers）”
在 Transformer 中，自注意力机制的核心是 `Softmax` 函数，要求所有 Token 的注意力权重之和必须为 $1$：
$$ \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V $$
当模型处理某些不需要太多上下文信息的 Token（例如标点符号、停用词）时，注意力机制仍然**被迫**要分配权值。随着模型变大，它学会了将这些“无处安放”的注意力集中投射到特定的 Token（如 `<s>` 或换行符）或特定的**隐藏特征维度**上。这些维度充当了“寄存器（Registers）”，吸收了多余的注意力，导致其激活值被不断累加放大。

#### 2. 绕过 LayerNorm 的信息传递通道
`LayerNorm` 会对每个 Token 的隐藏维度进行归一化：
$$ \text{LayerNorm}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \odot \gamma + \beta $$
如果模型想要在不同层之间传递一个强烈的、未经衰减的“全局指令”或“上下文标识”，它会发现：只要在一个特定的维度上产生一个极大的值，这个值就会主导方差 $\sigma^2$。这样，模型就可以利用这个巨大的维度作为控制其他特征缩放比例的“门控阀（Gating Mechanism）”。

#### 3. 为什么只有大模型（>6B）才会出现？
在小模型中，由于参数容量有限，模型倾向于将特征均匀分布在所有维度上以最大化表达能力。而当模型参数量足够大（>6B）时，模型具备了冗余的容量，它发现将少数维度专门“外包”出去作为全局控制器（Global Router）或偏置项（Bias），在数学上能够更有效地降低 Loss。这是一种典型的**涌现能力（Emergent Property）**。

---

### 二、 为什么微小的扰动或舍入（如量化）会产生致命影响？

在了解了异常值的成因后，就不难理解为什么在进行低比特量化（如 INT8）或引入微小噪声时，模型性能会呈现断崖式下降。

#### 1. 动态范围的“挤压效应”（Quantization Crushing）
这是量化过程中最核心的问题。假设我们有一个激活张量 $X$，其中 $99\%$ 的值都在 $[-1, 1]$ 之间，但有 $1\%$ 的异常维度值达到了 $100$。
当我们尝试使用最大绝对值（Max-Abs）将其对称量化到 INT8（范围 $[-127, 127]$）时，缩放因子（Scale）为：
$$ S = \frac{\max(|X|)}{127} = \frac{100}{127} \approx 0.787 $$
对于正常的特征值（如 $0.5$），其量化后的值为：
$$ X_{quantized} = \text{Round}\left(\frac{0.5}{0.787}\right) = \text{Round}(0.635) = 1 $$
反量化后变成了 $1 \times 0.787 = 0.787$。
**结论：** 异常值的存在拉大了缩放因子 $S$。为了保留这极少数的异常值，大部分正常特征的精度被严重压缩，甚至全部变成了 $0$。而这些正常特征恰恰包含了极其丰富的语义细节。

#### 2. 绝对误差的放大效应
在矩阵乘法 $Y = XW$ 中，异常维度（记为 $X_{outlier}$）虽然数量少，但在计算结果中占据主导地位。
如果由于硬件浮点舍入或量化截断，在 $X_{outlier}$ 上引入了哪怕 $1\%$ 的相对误差：
* 对于正常值 $1.0$，$1\%$ 的误差是 $0.01$。
* 对于异常值 $100.0$，$1\%$ 的误差是 $1.0$。
这会导致最终输出 $Y$ 的绝对误差被成百倍地放大，直接破坏了下一层的输入分布。

### 三、 工业界与学术界的解决方案

为了应对这种由于缩放导致的异常值，研究者们提出了多种巧妙的方法。

<details>
<summary><b>点击展开：LLM.int8() 的混合精度方案</b></summary>

`LLM.int8()` 发现，异常维度在特征维度（Channel）上是高度固定的（往往集中在少于 $1\%$ 的维度上）。
它的解决思路是**分离处理**：
1. 找出那些包含巨大激活值的列（异常维度）。
2. 将这部分 $<1\%$ 的异常维度保留为 FP16（16位浮点数）进行精确的矩阵乘法。
3. 将剩下 $>99\%$ 的正常维度进行 INT8 量化计算。
4. 最后将两部分的结果相加。
这种方法几乎实现了零性能损失，但引入了额外的内存操作开销。

```python
# LLM.int8() 伪代码示例
outlier_threshold = 6.0
outlier_mask = abs(X) > outlier_threshold

# 分离
X_outliers = X[:, outlier_mask] # 保持 FP16
X_normal = X[:, ~outlier_mask]  # 转换为 INT8

# 分别计算后合并
Y = matmul_fp16(X_outliers, W_outliers) + matmul_int8(X_normal, W_normal)
```
</details>

<details>
<summary><b>点击展开：SmoothQuant 的数学等价平滑方案</b></summary>

`SmoothQuant` 采用了另一种极其优雅的数学手段。既然激活值 $X$ 的方差太大很难量化，而权重 $W$ 的方差较小很容易量化，那为什么不把 $X$ 的难度转移给 $W$ 呢？
根据线性代数的结合律：
$$ Y = XW = (X \cdot \text{diag}(s)^{-1}) \cdot (\text{diag}(s) \cdot W) = \hat{X} \hat{W} $$
`SmoothQuant` 计算出一个每通道（Per-Channel）的缩放向量 $s$。
利用 $s$ 将 $X$ 中的异常值“除”小（使其变平滑），同时将 $W$ 对应维度“乘”大。
经过平滑后，$\hat{X}$ 和 $\hat{W}$ 都没有极端的异常值了，这时候就可以放心地对它们全局使用 INT8 量化，大大提升了推理速度（W8A8）。
</details>

<details>
<summary><b>点击展开：Attention Registers 显式提供 Sink</b></summary>

针对 Vision Transformers 论文（如 *Vision Transformers Need Registers*）中的发现，研究人员提出在序列的末尾或开头**显式地添加几个可学习的占位符 Token（Registers）**。
这相当于直接给模型提供了几个专门的“垃圾桶”。模型不需要再去扭曲正常的隐藏维度来存储无用信息，从而极大地减少了异常激活值的产生，使得模型生成的注意力图（Attention Maps）更加平滑和易于解释，同时也降低了量化难度。
</details>

### 🤖 Assistant

你的分析极其敏锐，精准地切中了 Transformer 架构底层设计引发“异常值（Outliers）”的数学本质。

你所描述的逻辑闭环非常完美：
因为 `Softmax` 强制所有注意力权重之和为 $1$，当注意力头想要“什么都不做（No-op）”时，它必须将概率质量（Probability Mass）倾泻到一个无关紧要的 Token 上。为了在经过 `LayerNorm` 的方差惩罚（除以 $\sigma$）后，依然能在注意力分数 $QK^T$ 中拉开足够的相对差距以接近概率 $1.0$，前一层的 `MLP` **被迫**去最大化某些特征维度的绝对值。

为了打破这个恶性循环，赋予 Attention 机制“不更新 Token 表示”的能力（即允许 Attention 输出接近于 $\vec{0}$），学术界和工业界探索了几种从根本上消解异常值需求的架构创新。

以下是实现这一目标的主要方法：

<details>
<summary><b>1. 引入零向量填充（Zero-Gated Attention / Null Node）</b></summary>

最直接的数学方法是打破 `Softmax` 强制分配给真实 Token 的概率和为 $1$ 的限制。
我们可以在 Keys 和 Values 矩阵中显式地追加一个固定的零向量（Null Node）。

假设原始的 Keys 和 Values 为 $K$ 和 $V$，我们追加一列全为零的向量 $\vec{0}_k$ 和 $\vec{0}_v$：
$$ \text{Attention}(Q, K, V) = \text{Softmax}\left(\frac{Q [K; \vec{0}_k]^T}{\sqrt{d}}\right) [V; \vec{0}_v] $$

**工作原理：**
当某个注意力头判断当前不需要提取任何上下文信息时，它可以将查询向量 $Q$ 与其他真实 Token 的点积变得非常小（甚至为负），从而使得与 $\vec{0}_k$ 的点积在 `Softmax` 中占据主导地位。
此时，分配给 $\vec{0}_v$ 的概率接近 $1.0$，整个 Attention 层的输出结果就会逼近 $\vec{0}$。
这样一来，残差连接 $x_{l+1} = x_l + \text{Attention}(...)$ 实际上就是 $x_{l+1} \approx x_l$，完美实现了“不更新表示”，且完全不需要 `MLP` 产生异常值来对抗 `LayerNorm`。
</details>

<details>
<summary><b>2. 使用 Sigmoid 替代 Softmax (Sigmoid Attention)</b></summary>

既然 `Softmax` 的归一化特性是罪魁祸首，另一种方案是彻底抛弃它，转而使用独立的 `Sigmoid` 激活函数。

$$ \text{Attention}(Q, K, V) = \text{Sigmoid}\left(\frac{QK^T}{\sqrt{d}}\right) V $$

**工作原理：**
在 `Sigmoid` 机制下，每个 Token 对其他 Token 的注意力分数是独立计算的，不再相互竞争。如果当前注意力头不想更新特征，它只需要让所有的注意力分数 $QK^T$ 都成为较大的负数，那么 `Sigmoid` 的输出就会全部趋近于 $0$。这同样允许 Attention 的输出整体收缩为零向量，从而避免了强制把注意力倾泻到无信息 Token 上的困境。
</details>

<details>
<summary><b>3. 显式添加注意力寄存器（Explicit Attention Registers）</b></summary>

这正是 *Vision Transformers Need Registers* 等论文提出的思路。既然模型总是要找个地方“倾倒”无用的注意力，不如我们主动为它提供“垃圾桶”。

在序列的输入端，除了正常的文本/图像 Token 外，拼接几个可学习的 `dummy_tokens`：
```python
# 假设 x 是经过 Embedding 的序列，形状为 [batch_size, seq_len, dim]
# registers 是可学习的参数，形状为 [batch_size, num_registers, dim]
x = torch.cat([registers, x], dim=1)

# 然后整体送入 Transformer 层
for layer in transformer_blocks:
    x = layer(x)

# 最终输出时，截掉 registers 即可
output = x[:, num_registers:, :]
```

**工作原理：**
这并未改变 Attention 的计算公式，但从系统层面缓解了压力。模型不需要再利用 `MLP` 在特定通道上制造极端值来强行突显标点符号或 `<bos>`，它只需要自然地将不需要的注意力分配给这些 `registers` 即可。这极大降低了特征分布的峰度（Kurtosis），使量化变得更加平滑。
</details>

<details>
<summary><b>4. 引入输出门控机制（Output Gating）</b></summary>

在 Attention 的输出端与残差流相加之前，增加一个由当前状态控制的门控网络。

$$ g = \text{Sigmoid}(\text{Linear}(x_l)) $$
$$ x_{l+1} = x_l + g \odot \text{Attention}(Q, K, V) $$

**工作原理：**
即使 Attention 层内部因为 `Softmax` 强制产生了一些不必要的输出，门控变量 $g$ 也可以根据当前 Token 的状态将其缩放为近乎 $\vec{0}$。这不仅赋予了模型直接忽略 Attention 结果的能力，也从梯度回传的角度降低了 `MLP` 去拟合极端异常值的激励（Incentive）。
</details>

### 总结

你指出的 **`LayerNorm` 缩放 -> `Softmax` 相对差异要求 -> `MLP` 异常值产生** 是一条非常核心的模型病态（Pathology）链条。

通过上述架构层面的微调（如引入零节点或替换激活函数），我们可以从数学根源上解耦“注意力权重的分配”与“特征更新的幅度”，从而在保持甚至提升模型表现力的同时，自然地消除这些对量化极度不友好的 Outliers。
