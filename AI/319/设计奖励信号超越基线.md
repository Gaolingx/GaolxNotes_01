# 设计奖励信号超越基线

### 🧑‍💻 User

在使用外部 LLM 提供奖励信号的 RLVR（Reinforcement Learning with Verifiable Rewards） 框架中，应如何设计奖励信号（可涉及 Prompt 工程、后训练策略等），才能使策略 LLM 的性能超越其奖励 LLM 在相同任务上的基线？

---

### 🤖 Assistant

在 RLVR (Reinforcement Learning with Verifiable Rewards) 框架中，让策略 LLM (Policy LLM) 的最终性能超越提供奖励信号的外部奖励 LLM (Reward LLM)，是当前大模型后训练（Post-training）领域的前沿挑战。这一目标的理论基础在于 **“验证比生成更容易”（P vs. NP 的直觉）**：Reward LLM 虽然自身生成正确答案的能力有限，但它能够高精度地辨别策略 LLM 生成的大量候选答案的好坏。

为了实现这一目标（即典型的 Weak-to-Strong Generalization 或超越教师模型），我们需要从**奖励信号设计**、**Reward LLM 的 Prompt 工程**以及**后训练强化学习策略**三个维度进行精心设计。

以下是实现该目标的系统性方案：

---

### 一、 核心奖励信号设计：从结果导向到过程导向

如果仅仅让 Reward LLM 评估最终结果（ORM, Outcome Reward Modeling），策略模型极易产生“奖励欺骗”（Reward Hacking），即通过错误的推理过程得出正确的结论。要超越基线，必须采用**过程奖励（PRM, Process-based Reward Modeling）**。

#### 1. 细粒度步骤验证 (Step-by-Step Verification)
不要让 Reward LLM 对整个回答给出单一打分，而是要求它对策略 LLM 生成的每一步进行验证。
*   设定状态动作对 $(s_t, a_t)$，其中 $a_t$ 是推理的一个中间步骤。
*   Reward LLM 仅输出三种信号：$+1$ (正确), $0$ (中立/无法确定), $-1$ (错误)。
*   整体序列的奖励不再是单一标量，而是累积奖励 $R = \sum_{t=1}^{T} r_t$。

#### 2. 多维度奖励融合 (Multi-dimensional Rubrics)
为了防止策略模型在特定维度上过度拟合，Reward LLM 需要从多个正交维度提供奖励：
*   **逻辑正确性 ($r_{logic}$):** 推理是否严密。
*   **格式遵循度 ($r_{format}$):** 是否符合目标任务的输出规范。
*   **探索惩罚/多样性 ($r_{div}$):** 惩罚过于重复的句式。
总奖励公式可以设计为：
$$R_{total} = \alpha r_{logic} + \beta r_{format} + \gamma r_{div}$$

---

### 二、 Reward LLM 的 Prompt 工程 (提升验证上限)

为了让 Reward LLM 能够指导策略 LLM 超越自身，我们需要通过 Prompt 工程**放大其验证能力**，使其在评估任务上的表现远超其直接生成任务的表现。

#### 1. 验证专用思维链 (Verification-CoT)
在让 Reward LLM 给出评分前，强制其先生成验证理由。这能显著降低 Reward LLM 的幻觉和误判率。

#### 2. 多数投票与自一致性 (Self-Consistency as Reward)
Reward LLM 单次评估可能存在噪声。对同一个推理步骤，让 Reward LLM 并发评估 $N$ 次（设置较高的 Temperature），然后取平均值或多数票作为最终奖励信号：
$$r_{final} = \frac{1}{N} \sum_{i=1}^{N} r_i^{(Reward\_LLM)}$$
这种方法利用了群体智慧，使得奖励信号的分布更加平滑和准确，从而提供比单一 Reward LLM 更高的天花板。

---

### 三、 后训练策略设计 (RLVR 核心循环)

有了高质量的奖励信号，关键在于如何更新策略模型 $\pi_\theta$。要超越基线，必须结合**搜索放大（Search Amplification）**和**迭代训练（Iterative Training）**。

#### 1. 拒绝采样微调 (Rejection Sampling Fine-Tuning, RSFT) - 冷启动
在引入 PPO 之前，先利用策略模型自身进行大规模探索：
1.  对于给定问题 $x$，让策略模型 $\pi_\theta$ 生成 $K$ 个不同的推理轨迹（例如 $K=64$）。
2.  使用外部 Reward LLM 对这 $K$ 个轨迹进行 PRM 评分。
3.  筛选出满分（或最高分）的轨迹 $y^*$。
4.  使用这些 $(x, y^*)$ 对策略模型进行 SFT（监督微调）。
*原理解析：通过生成 $K$ 个样本，我们利用了策略模型的方差，触及了比 Reward LLM 自身直接生成更好的解答，从而将搜索能力蒸馏到了策略模型中。*

#### 2. 带有 KL 散度约束的强化学习 (PPO/DPO)
在 RSFT 之后，使用 PPO 或 DPO 进行策略优化。为了防止策略模型“黑客式”地迎合 Reward LLM 的评估偏好，必须严格控制 KL 散度约束。
优化目标为最大化以下期望：
$$\mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta}[R(x, y) - \beta D_{KL}(\pi_\theta(\cdot|x) || \pi_{ref}(\cdot|x))]$$
*   **动态 KL 惩罚系数 ($\beta$):** 如果检测到策略生成的长度异常增加或出现特定重复模式（Reward Hacking 的前兆），自适应调大 $\beta$。

<details>
<summary>👉 展开查看：使用 Iterative DPO 替代 PPO 的优势</summary>

在某些 RLVR 场景下，使用 DPO（Direct Preference Optimization）比 PPO 更稳定。
做法：
1. 策略模型生成解答对 $(y_{win}, y_{lose})$。
2. Reward LLM 判定哪个更好。
3. 优化 DPO Loss：
$$ \mathcal{L}_{DPO}(\pi_\theta; \pi_{ref}) = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right] $$
这种方法不需要训练单独的 Value Network，降低了显存开销，并且在由外部 LLM 提供二元/偏好奖励时极其高效。
</details>

#### 3. 迭代式自我博弈与奖励刷新 (Iterative Self-Improvement)
这是超越 Reward LLM 性能的最关键机制：
*   **Iteration 0:** 策略 LLM 性能较弱，Reward LLM 可以轻松评估。
*   **Iteration 1:** 策略 LLM 经过 RLVR 训练，性能提升。此时，它生成的解题思路可能超出了 Reward LLM 的默认认知。
*   **策略调整:** 在后续迭代中，将策略 LLM 生成的**高置信度验证轨迹**（经过代码解释器或形式化验证器验证的轨迹）作为 In-context examples，反向 Prompt 给外部 Reward LLM，提升 Reward LLM 的评估能力。即：**让学生（Policy）探索出的新解法，拓宽老师（Reward）的眼界**。

---

### 四、 避免“超越失败”的关键工程细节

1.  **代码执行/形式化验证作为绝对基石 (Verifiable Anchors):**
    在数学或编程任务中，绝对不要 100% 依赖 Reward LLM。应当采用 `Reward = 规则验证 + LLM 启发式验证` 的混合模式。例如，最终答案必须能通过 Python `assert` 测试，只有在代码通过的前提下，才使用 Reward LLM 对代码的优雅度、时间复杂度进行打分。
2.  **对抗性负样本挖掘 (Adversarial Negative Mining):**
    刻意让策略模型生成看似正确但存在微小逻辑漏洞的解答（可以通过修改正确答案的中间步骤实现），测试 Reward LLM。如果 Reward LLM 给了高分，则人工干预或使用更强大的模型（如 GPT-4/Gemini 1.5 Pro）修正该奖励信号，防止策略模型在盲区中错误演化。
3.  **探索退火 (Exploration Annealing):**
    在 RL 初期，增大策略模型的 Temperature 鼓励探索；在 RL 后期，降低 Temperature 并增大 KL 惩罚，迫使模型在已知正确路径上收敛，防止过度探索导致的灾难性遗忘。

### 总结

要使策略 LLM 超越外部 Reward LLM，其本质是**将 Reward LLM 的“判别优势”通过大规模搜索（Best-of-N）转化为策略 LLM 的“生成优势”**。通过设计**步骤级过程奖励（PRM）**、使用 **Self-Consistency 放大奖励信号质量**，并在 PPO/DPO 中严格把控 **KL 惩罚以防止奖励欺骗**，最终通过多轮迭代，策略模型将内化出一套比外部奖励模型更强大的生成分布。
