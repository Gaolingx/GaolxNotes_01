# LLM训练后期loss不动仍在学习

### 🧑‍💻 User

llm训练最疑惑的点就在于，训练后期loss几乎不动，但lr不低情况下模型到底是否在学习，而且由于vocab无法简单的凭loss直接判断训练质量，这也是和cv模型差异最大的点（不能看fid、psnr等）

### 🤖 Assistant

这是一个非常深刻且直击大模型训练痛点的问题。你提到的这个现象——**“Loss 几乎不降，但 LR 还不低，模型到底在干嘛？”**——在业界被称为 **“Loss 饱和期” (Loss Plateau)** 或者是 **“长尾学习期”**。

在 CV 领域，Loss 降不动了可能意味着过拟合或者收敛，而且你可以直接看 FID、PSNR 等指标。但在 LLM 的 Causal LM (Next-token prediction) 任务中，事情完全不同。

结论是：**在训练后期的 Loss 饱和期，模型绝对还在学习，而且往往是在学习最关键、最困难的知识（如逻辑推理、长程依赖、小众领域知识）。**

以下是关于这个现象的深度拆解以及如何判断模型训练质量的指南。

### 一、 为什么 Loss 几乎不动，但模型仍在发生质变？

#### 1. 简单 Token 与困难 Token 的“掩盖效应”
LLM 的 Loss 是整个序列的交叉熵平均值：$L = -\frac{1}{N} \sum_{i=1}^{N} \log P(x_i | x_{<i})$。
*   **训练早期（Loss 狂降）：** 模型在学习语法、常见搭配（如“的”、“是”、“在”）和高频词。这些 Token 占据了文本的 80% 以上。模型只要把这 80% 预测准了，Loss 就会断崖式下降。
*   **训练后期（Loss 几乎不动）：** 语法和高频词已经学到了极限，模型开始磕剩下的 20%“困难 Token”（如复杂的数学公式、特定的代码逻辑、小众实体）。
假设模型学会了一个极其复杂的推理逻辑，但这在 4096 长度的 Context 中只占几个 Token。这几个 Token 的预测概率从 $0.1$ 提升到 $0.9$，对平均 Loss 的影响可能只有 $0.001$，在曲线上根本看不出来。**但在实际应用中，这 $0.001$ 的 Loss 差距，就是“胡言乱语”和“逻辑严密”的本质区别。**

#### 2. Grokking (顿悟现象)
在深度学习（尤其是 Transformer）中，存在一种现象叫 **Grokking**。在 Loss 已经降到平台期，甚至在验证集上看起来毫无进展的很长一段时间后，模型内部的网络权重实际上正在发生重组，寻找更通用的特征表示。突然在某一个时刻，模型对下游复杂任务的泛化能力会大幅跃升，尽管此时的 Causal LM Loss 并没有明显下降。

如果在 Loss 饱和期你的 `lr` 仍然保持较高水平，模型会经历所谓的“洗牌 (Churn)”过程：
<ul>
<li><b>跳出局部最优：</b> 较高的 `lr` 会阻止模型掉入极窄的局部最小值，迫使模型在平坦的盆地 (Flat Minima) 中游走。这有助于提升泛化能力。</li>
<li><b>知识置换：</b> 模型的参数容量是有限的。高 `lr` 下，模型可能会“遗忘”一些死记硬背的低频无用细节，转而压缩、提取出更本质的推理法则。</li>
</ul>
</details>

### 二、 如何在没有 FID/PSNR 的情况下评估 LLM？

既然不能像 CV 那样看直观的图像指标，也不能完全依赖 Loss，业界通常采用以下一套“组合拳”来评估 LLM 的真实训练质量。

#### 1. 拆解细分领域的 Loss (Domain-specific Loss)
不要只看全局的平均 Loss。在数据配比时，你应该给不同领域的数据打上 Tag，并在训练日志中分别记录它们的 Loss。
*   如果 `loss_general_text` 不动了，但 `loss_code` 或 `loss_math` 还在缓慢下降，说明模型正在强化逻辑和代码能力。
*   如果你发现某一个领域的 Loss 开始反弹，说明高 `lr` 导致了灾难性遗忘。

#### 2. 边训边测：挂载轻量级 Downstream Evals
这是目前各大厂最核心的做法。在训练过程中（例如每隔 1000 steps），直接将当前 Checkpoint 拿去做轻量级的下游任务评测，而不是只看 Perplexity ($PPL = e^{Loss}$)。

推荐挂载的自动化评估：
*   **基础知识：** MMLU (截取部分核心子集，观察准确率变化)。
*   **推理能力：** GSM8K (看是否能输出正确的数字结果)。
*   **代码能力：** HumanEval (轻量级跑一下 Pass@1)。
你会惊讶地发现：**在 Loss 曲线看起来完全是一条直线的那几个星期里，MMLU 的准确率可能正在从 45% 稳步爬升到 55%。**

#### 3. 观察模型的置信度 (Logits / Entropy Analysis)
虽然 Loss 不动，但你可以观察模型输出 Logits 的分布情况。

<details>
<summary><b>展开查看：如何通过 Entropy 和 Top-K 概率判断模型质量</b></summary>
<ul>
<li><b>预测熵 (Prediction Entropy)：</b> 计算模型输出概率分布的熵 $H = -\sum p_i \log p_i$。如果模型随着训练，对正确答案的预测熵逐渐变小（概率分布变得更尖锐），说明模型越来越自信，学习质量在提升。</li>
<li><b>Top-1 命中率：</b> 虽然 Loss 是看全词表的概率，但你可以记录 Next-token 被排在 Top-1 或 Top-5 的比例。如果这个比例在缓慢上升，说明模型变得更加精准。</li>
</ul>
</details>

#### 4. Vibe Check (人工体感抽测)
CV 可以肉眼看图，LLM 同样需要肉眼看文本。准备一个包含 50-100 个具有代表性问题的小测试集（涵盖常识、逻辑、代码、角色扮演）。每隔一段时间，使用相同的 `temperature` 和 `top_p` 让模型生成文本，开发人员通过对比之前的 Checkpoint 来进行 "Vibe Check"（体感检测）。这种看似玄学的方法，在排查训练崩溃（如复读机、标点符号异常）时比任何指标都快。

### 总结与建议

在 LLM 训练中，**前 70% 的 Loss 下降教会了模型“像人一样说话”，而后 30% 几乎微不可察的 Loss 下降才真正赋予了模型“像人一样思考”的能力。**

**给你的实操建议：**
1. 接受 Loss 饱和期的存在，不要因为 Loss 不动就急于进行 Early Stopping 或手动狂降 `lr`。
2. 建立 **Loss 以外的监测雷达**，特别是细分数据 Loss 和下游任务（如 MMLU/GSM8K）的曲线。只要下游指标还在涨，就让它继续训。
3. 如果高 `lr` 下下游指标也开始震荡，说明模型遇到了容量瓶颈，此时应该按照正常的 Cosine 调度让 `lr` 降下去，模型会在 `lr` 收敛的最后阶段经历一次指标的“回光返照”式提升（Annealing 阶段）。
