# E4M3用于前向E5M2用于反向

### 🧑‍💻 User

The FP8 (floating point 8) with two different FP8 encodings: E4M3 (4-bit exponent and 3-bit mantissa) and E5M2 (5-bit exponent and 2-bit mantissa).The potential floating points that can be represented in the E4M3 format are in the range -448 to 448, whereas in the E5M2 format, as the number of bits of the exponent increases, the range increases to -57344 to 57344 - but with a loss of precision because the number of possible representations remains constant. So why the E4M3 is best suited for the forward pass, and the second version is best suited for the backward computation?

### 🤖 Assistant

The decision to use **E4M3** for the forward pass and **E5M2** for the backward pass in FP8 mixed-precision training comes down to a fundamental trade-off between **precision** (resolution) and **dynamic range** (span of representable values), perfectly matched to the statistical properties of deep learning workloads.

Here is the breakdown of why this dual-format approach works so well.

### 1. Why E4M3 is Best for the Forward Pass

During the forward pass, the neural network computes outputs using **weights** and **activations**. 

*   **Need for High Precision:** Weights and activations dictate the actual function the neural network is computing. If you lose precision here, the model's output diverges significantly from what it should be, leading to an inaccurate loss calculation and degrading model quality. E4M3 has a 3-bit mantissa, giving it double the precision of E5M2.
*   **Narrow, Predictable Range:** Modern neural networks use normalization techniques (like LayerNorm or BatchNorm). These techniques constrain weights and activations to a very well-behaved, narrow distribution, typically centered around $0$ (e.g., a standard normal distribution $\mathcal{N}(0,1)$). 
*   **Conclusion:** Because forward-pass values rarely spike to massive extremes, the extended range of E5M2 ($[-57344, 57344]$) is wasted. Instead, E4M3 safely captures the required range ($[-448, 448]$) while providing the vital precision needed to accurately differentiate between closely clustered activation values.

### 2. Why E5M2 is Best for the Backward Pass

During the backward pass, the network calculates **gradients** (the derivatives of the loss with respect to the weights). Gradients behave very differently from weights and activations.

*   **Need for Massive Dynamic Range:** Gradients span many orders of magnitude. The gradients for layers near the output might be large, while gradients for layers near the input might be astronomically small (the infamous vanishing gradient problem). If the exponent range is too small, small gradients will round down to $0$ (**underflow**), halting the learning process. The 5-bit exponent in E5M2 provides a vastly wider dynamic range to prevent this.
*   **Tolerance to Lower Precision:** Optimization algorithms like Stochastic Gradient Descent (SGD) and Adam are inherently noisy. They do not need point-perfect precision to find the minimum of the loss landscape; they only need a generally correct direction and magnitude. Therefore, sacrificing one bit of the mantissa (moving from 3 bits to 2 bits) does not noticeably harm the optimization process.
*   **Conclusion:** In the backward pass, preventing gradient underflow is much more important than absolute numerical precision. E5M2 trades a bit of precision to guarantee that both tiny and huge gradients are preserved.

<summary><b>Deep Dive: FP8 Bit Layouts and Mathematical Properties</b></summary>

For a floating-point number, the value is calculated using the formula:
$$(-1)^{\text{sign}} \times 2^{\text{exponent} - \text{bias}} \times \left(1 + \frac{\text{mantissa}}{2^{\text{mantissa\_bits}}}\right)$$

Here is how the two FP8 formats compare at the bit level:

| Feature | E4M3 (Forward Pass) | E5M2 (Backward Pass) |
| :--- | :--- | :--- |
| **Bit Layout** | 1 Sign, 4 Exponent, 3 Mantissa | 1 Sign, 5 Exponent, 2 Mantissa |
| **Exponent Bias** | $7$ | $15$ |
| **Max Value** | $448$ | $57344$ |
| **Min Non-Zero Value** | $2^{-9} \approx 0.00195$ | $2^{-16} \approx 0.0000152$ |
| **NaN / Inf** | Supports NaN, usually no Inf | Follows IEEE 754 (Supports Inf & NaN) |

As shown in the table, the **Min Non-Zero Value** for E5M2 is exponentially smaller than that of E4M3. If gradients in the backward pass fall to a value like $0.0005$, E4M3 would round them to $0$ (underflow), whereas E5M2 can represent them easily.

### Summary
To put it simply: **Forward passes care about accuracy (E4M3), while backward passes care about capturing extremes (E5M2).** By combining them, hardware accelerators (like NVIDIA's Hopper architecture) can perform $8$-bit training that achieves the same model accuracy as $16$-bit or $32$-bit training, but at twice the speed and half the memory footprint.
