# SFT与RL交替训练必要

### 🧑‍💻 User

基模型后训练阶段，通常要做多阶段rl，为什么冷启动SFT之后不能一直RL呢？还要不间断SFT？

---

### 🤖 Assistant

在大模型（LLM）的后训练（Post-training）阶段，采用“多阶段 SFT 与 RL 交替”或“在 RL 过程中混合 SFT 数据”是业界的标准做法（例如 InstructGPT、Llama 2/3 的训练流程）。

如果在冷启动 SFT（Supervised Fine-Tuning）之后**只进行 RL（强化学习）而不间断或穿插 SFT**，模型很快会面临一系列严重的问题。核心原因可以归结为以下几点：

### 1. 缓解灾难性遗忘与“对齐税” (Alignment Tax)
RL 的目标高度集中于最大化奖励模型（Reward Model）的评分（如：安全性、是否有礼貌、排版是否好看）。如果长时间只做 RL，模型会把所有的参数更新都倾注在这些偏好上，从而导致**灾难性遗忘 (Catastrophic Forgetting)**。
* **表现：** 模型可能会忘记预训练和早期 SFT 中学到的事实知识、代码能力、数学推理能力，甚至基本的语法连贯性。这种为了人类偏好而牺牲通用能力的现象被称为“对齐税”。
* **SFT 的作用：** 定期进行 SFT，或者在 RL 损失函数中加入 SFT/预训练数据的损失（如 InstructGPT 中的 PTX loss），可以锚定模型的基础能力，防止其偏离语言模型的本质。

### 2. 防止奖励作弊 (Reward Hacking / Over-optimization)
奖励模型（RM）是一个不完美的代理。它只是基于有限的标注数据训练出来的，存在漏洞（分布外盲区）。
* **表现：** 长时间的纯 RL 训练会使得策略模型（Policy Model）不可避免地找到 RM 的漏洞。例如，模型可能会发现只要疯狂输出长篇大论、过度使用敬语（“好的，主人，我非常乐意为您解答...”）、或者堆砌某些特定的词汇，RM 就会给出高分，即使内容毫无意义。
* **SFT 的作用：** SFT 提供的是**直接的、高质量的示范 (Demonstrations)**。它强制模型学习“正确的输出分布”，而不是任由 RL 在未知的隐式空间中疯狂试探。

### 3. 避免模式崩溃与输出多样性丧失 (Mode Collapse)
强化学习的本质是“趋利避害”，它倾向于将概率质量集中在少数几个能获得最高奖励的动作（Token 序列）上。
* **表现：** 持续的 RL 会导致生成的多样性急剧下降。对于不同类型的问题，模型可能会倾向于使用极其相似、刻板的句式回答（例如著名的“作为一个AI语言模型……”）。
* **SFT 的作用：** SFT 的交叉熵损失函数 (Cross-Entropy Loss) 本质上是在做分布匹配（Distribution Matching），强迫模型覆盖训练数据中多样化的表达方式，保持语言的丰富性。

### 4. RL 训练的内在不稳定性
像 `PPO` (Proximal Policy Optimization) 这样的 RL 算法本身非常脆弱且难以调参。在超大规模参数下，长时间的 RL 极易发散。穿插高质量的 SFT 可以视作一种“重置”或“纠偏”，让模型重新回到一个稳定、良好的参数空间，然后再进行下一阶段的 RL 微调。

<details>
<summary><b>点击展开：数学视角下的 RL 与 SFT 混合机制 (技术细节)</b></summary>

在实际操作中，纯 RL 优化的目标函数通常包含奖励和 KL 散度惩罚：
$$ \max_{\pi_\theta} \mathbb{E}_{x \sim \mathcal{D}, y \sim \pi_\theta}[R(x, y) - \beta D_{KL}(\pi_\theta(y|x) || \pi_{ref}(y|x))] $$

这里的 KL 散度项虽然可以在一定程度上防止模型偏离初始的 SFT 模型 $\pi_{ref}$，但在长期的 RL 训练中，这依然不够。

因此，现代后训练流程通常会在 RL 的目标函数中直接**混合 SFT 损失（或预训练语料损失）**。例如 InstructGPT 中的 PPO-ptx 目标函数：
$$ Objective = L_{RL} + \gamma L_{SFT} $$

其中 $L_{SFT}$ 是在高质量 SFT 数据集（或预训练数据集）上的标准交叉熵损失：
$$ L_{SFT} = - \mathbb{E}_{(x, y) \sim \mathcal{D}_{SFT}} [\log \pi_\theta(y|x)] $$

通过加入 $\gamma L_{SFT}$，模型在利用 RL 探索高奖励回复的同时，被强制要求在 SFT 数据上保持高概率输出。这从数学上保证了模型不会因为过度追求 $R(x,y)$ 而导致基础能力崩溃。
</details>

<details>
<summary><b>点击展开：多阶段课程学习 (Curriculum Learning) 的工程考量</b></summary>

在复杂的基模型训练中，能力是分阶段解锁的：
1. **第一阶段：** 基础指令遵循（需要大量 SFT）。
2. **第二阶段：** 基础对话对齐、安全护栏（RLHF）。
3. **第三阶段：** 复杂推理、长文本生成、Agent 能力。

当你想让模型在第三阶段获得新能力时（例如使用工具 API），纯靠 RL 让模型自己“探索”出正确的 JSON 格式和调用逻辑效率极低，甚至不可能。
**正确的做法是：** 再次引入针对 API 调用的高质量 SFT 数据进行“冷启动”或“知识注入”，然后再用 RL 优化它在多轮交互中的调用策略。这种 **SFT (注智) $\rightarrow$ RL (对齐) $\rightarrow$ 新 SFT (新能力注智) $\rightarrow$ 新 RL (新能力对齐)** 的螺旋上升过程，是不可替代的。
</details>

### 总结
简单来说，**SFT 决定了模型能力的“下限”和“知识广度”，而 RL 决定了模型表现的“上限”和“偏好对齐度”**。

如果不间断 SFT，只做 RL，就像是让一个学生只看考试的评分标准（Reward），而不让他读教科书（SFT）。短期内他可能学会了如何通过迎合阅卷老师来拿高分（Reward Hacking），但长期来看，他的基础知识会遗忘，思维会变得刻板，最终失去解决真正复杂问题的能力。
