# 过滤空prompt和空response样本

### 🧑‍💻 User

空 prompt、空 response的sft样本到底应该过滤吗？还是应该只过滤空 response样本？

---

### 🤖 Assistant

在绝大多数大语言模型（LLM）的指令微调（SFT, Supervised Fine-Tuning）场景中，**强烈建议将「空 prompt」和「空 response」的样本全部过滤掉**。

这两种样本通常是由于数据抓取、清洗或格式转换过程中的错误产生的脏数据。保留它们会对模型的表现产生负面影响。

以下是具体的原理解析：

### 1. 为什么必须过滤「空 Response」样本？（绝对要过滤）

在 SFT 阶段，模型学习的目标是给定输入 $X$（Prompt），最大化输出 $Y$（Response）的概率，即拟合条件概率 $P(Y|X)$。

如果 Response 为空，意味着模型在接收到某个 Prompt 后，目标输出直接就是结束符（如 `<EOS>`、`<|endoftext|>`）。
*   **致命后果（模型变“懒”）：** 模型会学习到“面对问题直接闭嘴”的映射关系。如果这类数据较多，模型在推理时会出现截断问题（early stopping），即使遇到正常的 Prompt，也可能直接输出 `<EOS>` 从而生成空白回复。
*   **指令对齐失败：** SFT 的核心是教模型“如何回答问题”，空 Response 完全违背了这一目标，没有任何训练价值。

### 2. 为什么也应该过滤「空 Prompt」样本？（99%的情况下要过滤）

虽然空 Prompt 看上去只是让模型“自由发挥”，但它在标准 SFT 中同样有害。

*   **破坏条件生成逻辑：** SFT 是**条件生成**。如果 Prompt 为空，条件概率 $P(Y|X)$ 就退化成了无条件概率 $P(Y)$。模型相当于退回到了预训练（Pre-training）状态，只是在盲目地背诵这段 Response 文本，而不是学习“指令-回复”的映射关系。
*   **引发幻觉或自言自语：** 如果模型在空 Prompt 下被训练生成大段文本，在实际应用中，当用户输入极短或存在歧义的 Prompt 时，模型极易被触发进入“自言自语”模式，输出大量与用户意图无关的内容。
*   **数据质量污染：** 通常情况下，正常的业务逻辑中不存在没有前提输入的高质量回答。空 Prompt 往往意味着这本来是一个优质的多轮对话或长文本QA，但因为代码 bug 导致 Question 字段丢失了。这属于残缺数据，学习残缺数据毫无意义。

<details>
<summary><b>💡 极少数可以保留「空 Prompt」的边缘特例（点击展开）</b></summary>

在极少数特定架构或非标准 SFT 任务中，空 Prompt 可能有意义，但这些**不属于常规的 LLM 指令微调**：

1.  **多模态模型（VLM）：** 如果你的模型是多模态的，有时候文本 Prompt 虽然为空，但输入包含了一张图片（Image），此时的期望输出是图片的 Caption。这种情况下“文本空 Prompt”是合理的。
2.  **无条件生成任务混合：** 有些研究人员会在 SFT 阶段混入少量的高质量文档（类似继续预训练 Continuation Pre-training），此时这些文档没有 Prompt，只有正文。但这通常需要特定的 loss mask 策略，且不叫严格意义上的 SFT 样本。
3.  **测试模型的无条件分布：** 极其少见，主要用于学术分析模型在没有任何提示词时的先验偏好。

</details>

<details>
<summary><b>🛠️ SFT 数据清洗的最佳实践建议（点击展开）</b></summary>

为了保证 SFT 质量，建议在数据预处理 Pipeline 中加入以下规则：

1.  **非空校验：** 剔除 `prompt.strip() == ""` 或 `response.strip() == ""` 的数据。
2.  **长度阈值过滤：** 仅仅非空还不够。如果 Prompt 或 Response 只有 1-2 个无意义的字符（如 `"ok"`、`"1"`、`"?"`），也应该被过滤。通常会设置一个最小字符数或最小 Token 数阈值。
3.  **无效字符过滤：** 有些空数据表现为一堆空格、换行符 `\n\n\n` 或 HTML 标签。在判定是否为空之前，必须先做 `strip()` 和基础的清洗。
4.  **多轮对话中的空回合：** 在多轮对话数据（如 OpenAI 的 ChatML 格式）中，如果中间某一轮的 User 或 Assistant 内容为空，应丢弃整条对话历史或截断该空轮次之后的内容，因为上下文逻辑已经断裂。

</details>

### 总结
**不要犹豫，两者都过滤掉。** SFT 数据遵循 "Garbage in, garbage out"（垃圾进，垃圾出）的原则，数据的质量远比数量重要。空样本不仅浪费算力，还会破坏模型遵循指令的能力。
