# SFT阶段LoRA与全参微调学习率设置

### 🧑‍💻 User

sft阶段lora微调和全参微调峰值lr如何设置？

---

### 🤖 Assistant

在SFT（指令微调）阶段，设置合适的峰值学习率（Peak Learning Rate）是模型训练成功的关键。LoRA微调和全参微调（Full Parameter Fine-tuning）由于更新的参数量和优化空间完全不同，它们的学习率设置也有显著差异。

核心原则是：**全参微调需要较低的学习率以防止灾难性遗忘，而LoRA微调需要较高的学习率以推动少量参数快速拟合。**

以下是具体的设置建议：

### 1. 全参微调 (Full Parameter Fine-Tuning)
在全参微调中，模型的所有权重都会被更新。因为预训练模型已经具备了很好的基础能力，过大的学习率会破坏模型原有的知识表达（即灾难性遗忘）。

*   **经验范围**：$5 \times 10^{-6}$ 到 $2 \times 10^{-5}$
*   **常见默认值**：$1 \times 10^{-5}$ 或 $2 \times 10^{-5}$
*   **设置建议**：
    *   **模型越大，学习率越小**：例如 7B 模型可以使用 $2 \times 10^{-5}$，而 70B 模型可能需要降低到 $1 \times 10^{-5}$ 甚至 $5 \times 10^{-6}$。
    *   **数据量与质量**：如果 SFT 数据集非常大（几十万条以上）且质量高，可以稍微调大一点点；如果数据集很小，建议保持在保守的低学习率。

### 2. LoRA 微调 (LoRA Fine-Tuning)
在 LoRA 中，我们冻结了原始权重，只训练旁路的低秩矩阵（A 和 B）。由于初始状态下矩阵 B 通常初始化为 0（使得初始输出与预训练模型完全一致），且可训练参数极少（通常不到总参数的 1%），因此需要更大的学习率来驱动这些参数。

*   **经验范围**：$1 \times 10^{-4}$ 到 $5 \times 10^{-4}$
*   **常见默认值**：$2 \times 10^{-4}$ 或 $3 \times 10^{-4}$
*   **设置建议**：
    *   对于大多数主流开源模型（如 Llama 3, Qwen 2, ChatGLM 等），$2 \times 10^{-4}$ 是一个非常安全且高效的起点。
    *   如果发现 Loss 下降过于缓慢，可以尝试提升至 $3 \times 10^{-4}$ 或 $5 \times 10^{-4}$。
    *   如果发现 Loss 出现震荡或发散（NaN），应立即降低学习率至 $1 \times 10^{-4}$ 以下。

---

除了微调方式外，实际设置时还需要结合以下几点进行动态调整：

#### 1. 批次大小 (Global Batch Size)
根据深度学习中的线性缩放法则（Linear Scaling Rule），学习率通常需要与 Batch Size 匹配：
*   **Global Batch Size 较小**（如 16-64）：使用上述推荐范围的**下限**。
*   **Global Batch Size 较大**（如 128-512）：使用上述推荐范围的**上限**。
*(注：Global Batch Size = Micro Batch Size $\times$ 梯度累加步数 $\times$ GPU数量)*

#### 2. LoRA 的 $r$ 与 $\alpha$ 设置
LoRA 的权重更新包含一个缩放因子：$\frac{\alpha}{r}$。
*   如果你的设置是 $\alpha = 2r$（例如 $r=8, \alpha=16$），缩放因子恒定，学习率按正常推荐值即可。
*   如果你改变了比例（例如 $\alpha = r$），你可能需要相应地**调大**学习率来补偿缩放因子的减小。

#### 3. 学习率调度器 (LR Scheduler) 与 预热 (Warmup)
峰值学习率是指预热阶段结束时的最高学习率。
*   **Scheduler**：SFT 阶段强烈建议使用 **余弦退火（Cosine Decay）** 调度器，让学习率在训练后期平滑下降到接近 0（或峰值的 10%）。
*   **Warmup**：必须设置预热期。通常设置为总训练步数的 **3% 到 5%**，或者固定的 **50~100步**。这可以防止训练初期出现极大的梯度截断或 Loss 爆炸。

</details>

### 总结建议（如何开始实验）

如果你正在准备跑第一次实验，建议采用以下基准配置（假设使用 AdamW 优化器，Global Batch Size $\approx 128$）：

| 微调方式 | 推荐 Peak LR | Warmup 比例 | LR Scheduler | 权重衰减 (Weight Decay) |
| :--- | :--- | :--- | :--- | :--- |
| **LoRA** | $2 \times 10^{-4}$ | 5% | Cosine | 0.01 到 0.05 |
| **全参 (Full)** | $1 \times 10^{-5}$ | 3% | Cosine | 0.0 到 0.01 |

开始训练后，密切观察前 100 步的 Loss 曲线。如果预热结束后 Loss 没有明显下降趋势，说明学习率偏小；如果预热一结束 Loss 就剧烈波动甚至上升，说明学习率过大。

### 🧑‍💻 User

如何根据不同lora rank设置alpha和lr？为什么需要学习率缩放，lora和bs分别如何计算？

---

### 🤖 Assistant

在理解如何根据 LoRA 的 Rank ($r$) 设置 $\alpha$ (Alpha) 和学习率 (LR) 之前，我们需要先明白它们在底层计算中是如何相互作用的。

简而言之：**通过合理设置 $\alpha$ 与 $r$ 的比例，你可以尽量避免在改变 $r$ 时去重新寻找最佳的学习率。**

以下是详细的设置规则、缩放原因以及计算方法。

---

### 1. 如何根据不同的 LoRA Rank 设置 $\alpha$ 和 LR？

在 LoRA 微调中，前向传播的计算公式大致为：
$h = W x + \frac{\alpha}{r} B A x$

这里，$\frac{\alpha}{r}$ 是一个**缩放因子 (Scaling Factor)**。它直接乘在 LoRA 注入的权重更新上。

#### 核心设置规则：保持 $\frac{\alpha}{r}$ 的比值恒定
业界最佳实践是：**确定一个你认为好的比例（通常是 1 或 2），并在调整 $r$ 时保持这个比例不变。**

*   **常见基准**：设置 $\alpha = r$ （比例为1） 或 $\alpha = 2r$ （比例为2）。
*   **举例**：
    *   如果你用 $r = 8$，可以设置 $\alpha = 16$ （比例为 2）。此时最佳 LR 假设为 $2 \times 10^{-4}$。
    *   如果你想增加模型容量，将 $r$ 提升到 $64$，你应该将 $\alpha$ 设置为 $128$。
    *   **结果**：因为 $\frac{16}{8} = \frac{128}{64} = 2$，缩放因子没变。此时，你的**学习率 LR 不需要做任何改变**，依然可以使用 $2 \times 10^{-4}$。

#### 如果比例改变了，LR 怎么设？
如果你因为某些原因（或使用了旧的代码库默认值）改变了 $\frac{\alpha}{r}$ 的比例，你必须反向调整学习率来补偿。
*   **公式**：$LR_{new} \approx LR_{base} \times \frac{(\alpha/r)_{base}}{(\alpha/r)_{new}}$
*   **举例**：你的基准是 $r=8, \alpha=16, LR=2 \times 10^{-4}$（比例为2）。现在你改成了 $r=64, \alpha=16$（比例变成了0.25，缩小了8倍）。为了获得同样的更新幅度，你需要把学习率**放大8倍**，即 $LR_{new} = 1.6 \times 10^{-3}$。*(注：过大的学习率可能导致 Adam 优化器不稳定，所以强烈建议保持 $\alpha$ 和 $r$ 同步缩放，而不是去改 LR)*。

---

### 2. 为什么需要学习率缩放？

缩放的本质是为了**控制参数更新的步长和方差**，使得超参数具有可复用性。

#### 针对 LoRA 的缩放（为什么要有 $\frac{\alpha}{r}$？）
当我们将秩 $r$ 增大时，矩阵 $A$ 和 $B$ 的维度变大。在计算内积 $BAx$ 时，参与累加的项变多了。
如果没有 $\frac{1}{r}$ 这个缩放项，随着 $r$ 的增大，输出的方差会成比例变大，这就导致你每次改变 $r$，网络中的激活值范围就会剧变，你必须重新寻找一个极其微小的学习率来防止梯度爆炸。
原作者引入 $\alpha$ 并除以 $r$，是为了在初始化和训练时**解耦 $r$ 和学习率**。只要 $\frac{\alpha}{r}$ 恒定，你调参时就可以随意尝试不同的 $r$，而不用担心学习率失效。

#### 针对 Batch Size 的缩放（为什么 BS 变大 LR 也要变？）
梯度下降的核心是利用当前 Batch 的数据来估计全局梯度的方向。
*   **Batch Size 小**：相当于管中窥豹，估计出来的梯度方向包含很多噪声。为了防止被噪声带偏，走错方向，步子必须迈小一点（**较小的 LR**）。
*   **Batch Size 大**：样本多，噪声被平均掉了，计算出的梯度方向非常准确，非常接近真实的全局最优方向。既然方向对了，就可以放心大胆地迈大步子（**较大的 LR**），以加快收敛速度。

---

### 3. LoRA 和 Batch Size (BS) 的缩放具体如何计算？

#### LoRA 缩放的实际应用
在 HuggingFace 的 `peft` 库中，你只需要在 `LoraConfig` 中直接传入这两个参数：
```python
from peft import LoraConfig

# 比例为 2 的配置，适用于 LR = 2e-4
config = LoraConfig(
    r=16, 
    lora_alpha=32, 
    # ... 其他参数
)
```
**计算总结**：实际更新步长 $\propto LR \times \frac{\alpha}{r}$。控制后面那一项不变，LR 就不变。

#### Batch Size 的缩放计算法则

假设你在 Global Batch Size = 32 时，调出了一个最佳学习率 $LR_{base}$。现在你有更多的 GPU，把 Global Batch Size 提升到了 128。

<details>
<summary><b>点击展开：SGD 优化器 vs AdamW 优化器的不同缩放法则</b></summary>

**1. 线性缩放法则 (Linear Scaling Rule) —— 主要适用于 SGD**
学术界最经典的法则（由 Facebook 在 2017 年提出）：
$$LR_{new} = LR_{base} \times \frac{BS_{new}}{BS_{base}}$$
*   *计算*：BS 从 32 扩大到 128（扩大 4 倍），LR 也扩大 4 倍。

**2. 平方根缩放法则 (Square Root Scaling) —— 更适用于 AdamW (LLM常用)**
由于大型语言模型 SFT 几乎全部使用 `AdamW` 优化器，Adam 本身会根据梯度的二阶矩（方差）对学习率进行自适应除法归一化。因此，它对 BS 的变化不如 SGD 那么敏感。
针对 AdamW，经验上更推荐使用平方根缩放（或介于不变与平方根之间）：
$$LR_{new} = LR_{base} \times \sqrt{\frac{BS_{new}}{BS_{base}}}$$
*   *计算*：BS 从 32 扩大到 128（扩大 4 倍），$\sqrt{4} = 2$，因此 LR 扩大 **2 倍** 即可。

</details>

**LLM SFT 中 Batch Size 与 LR 的实际计算建议 (AdamW)：**
1.  **基准**：假设在 `Global BS = 64` 时，你的最佳 LoRA LR 是 $2 \times 10^{-4}$。
2.  **缩小 BS**：如果你算力有限，只能跑到 `Global BS = 16` (缩小 4 倍)，建议将 LR 减半，设置为 $1 \times 10^{-4}$。
3.  **增大 BS**：如果你的 `Global BS = 256` (扩大 4 倍)，建议将 LR 翻倍，设置为 $4 \times 10^{-4}$（注意监控，如果起步阶段 Loss 震荡，可以回退到 $3 \times 10^{-4}$）。

### 🤖 Assistant

针对 500 万（5M）样本这种**极其庞大且多样化**的首次 SFT 数据集，使用标准 LoRA 进行微调是一个非常有挑战性的任务。

在这么大的数据量和广泛的数据分布（涵盖通用对话、安全、代码、理科、CoT、工具调用等）下，模型需要学习和对齐的知识非常多。虽然 LoRA（$r=128, \alpha=256$）能节约显存，但其表达能力（Capacity）相比全参微调依然有限。因此，超参数的设置必须为了**最大化知识吸收率**和**保持训练稳定性**而设计。

以下是针对此特定场景的超参数设置建议及详细解析：

### 1. 核心超参数配置推荐

| 超参数 | 推荐设置 | 说明 |
| :--- | :--- | :--- |
| **Effective Batch Size** | **512 到 1024** | 5M数据量极大，需要大 BS 保证梯度下降方向稳定，加速训练。 |
| **Train Modules (Target)** | **`all-linear` (所有线性层)** | **必须！** 仅微调 Q/V 已无法支撑 5M 多样化数据的拟合。 |
| **Peak Learning Rate** | **$1 \times 10^{-4}$ 到 $2 \times 10^{-4}$** | 配合大 BS，虽然 $\alpha/r=2$，但由于模块全开且数据杂，不宜过高。 |
| **Scheduler** | **Cosine (带最小 LR 截断)** | 余弦退火，最后不要降到 0，降到峰值的 10%。 |
| **Warmup Steps** | **2000 到 3000 steps** | 取决于总 step，建议约占总步数的 3%-5%。 |
| **Epochs** | **1 到 2 (最好是 1)** | 5M 数据量过大，通常 1 个 Epoch 足以收敛，多跑容易过拟合在简单样本上。 |

---

### 2. 参数设置详细解析与理由

#### A. Effective Batch Size (全局批次大小)
对于 5M 的数据量，较小的 Batch Size（如 64 或 128）会导致训练步数极多，不仅耗时漫长，而且在面对复杂数据分布时，小批次的梯度噪声大，容易导致模型在不同任务（如刚学完代码又去学安全）之间发生“灾难性遗忘”或震荡。
*   **建议设置**：`eff_bs = 512` 或 `1024`。
*   **计算逻辑**：假设 `eff_bs = 512`，5,000,000 条数据 1 个 Epoch 约为 9765 个 Global Steps。这是一个非常合理的优化步数范围。
*   **实现方式**：利用梯度累加（Gradient Accumulation）来实现大有效批次。例如，8 卡训练，单卡 `micro_bs=8`，则 `grad_accum_steps = 512 / (8 * 8) = 8`。

#### B. Train Modules (目标模块)
**这是此场景下最关键的设置之一。**
早期 LoRA 的默认设置通常只微调 Attention 层的 `q_proj` 和 `v_proj`。但这对于 5M 级别、包含逻辑推理 (CoT) 和代码 (SWE) 等深度知识的数据集来说，**参数容量绝对不够，会导致严重欠拟合**。
*   **建议设置**：微调所有线性层 (`all-linear`)。
    *   在 LLaMA/Qwen 架构中，通常包括：`q_proj`, `k_proj`, `v_proj`, `o_proj`, `gate_proj`, `up_proj`, `down_proj`。
*   **理由**：大量论文和实践表明，微调 MLP 层（`gate/up/down`）对于大模型吸收事实性知识和提升逻辑推理能力至关重要。虽然微调所有层会增加部分显存和计算量（相较于仅 Q/V），但相比全参微调仍节省极大，且效果远超仅微调 Attention。
*   **代码示例 (PEFT)**:
    ```python
    lora_config = LoraConfig(
        r=128,
        lora_alpha=256,
        target_modules="all-linear",
        # ...
    )
    ```

#### C. Peak Learning Rate (峰值学习率)
你设定的 $r=128, \alpha=256$，缩放因子 $\frac{\alpha}{r} = 2$。按照标准 LoRA 推荐，LR 本可以设为 $2 \times 10^{-4}$ 到 $3 \times 10^{-4}$。
*   **为什么建议稍微保守（$1 \times 10^{-4}$ 到 $1.5 \times 10^{-4}$）？**
    1.  你开启了 **`all-linear`**，可训练参数量比普通的 Q/V LoRA 多了数倍（约占原模型的 2%-5%），参数量的增加需要适当降低一点学习率。
    2.  数据分布广，混合了大量长文本和推理样本（CoT/Code），梯度方差可能较大，保守一点能避免 NaN。
*   **建议**：以 $1.5 \times 10^{-4}$ 作为首选。如果观察到前 1000 步 Loss 下降极慢，可以 stop 并在重启时上调至 $2 \times 10^{-4}$。

#### D. Scheduler (调度器) 与 Warmup
*   **Scheduler**：选择 **Cosine**。这是最稳妥的。但在配置文件中，建议设置 `lr_scheduler_kwargs` 将最终的最小学习率 (`eta_min` 或 `min_lr`) 设置为峰值的 10%（例如 $1.5 \times 10^{-5}$）。不要让学习率降到绝对的 0，这样在训练末期模型依然能微调细节。
*   **Warmup**：对于超大数据集，预热步数需要足够长。如果总 step 是 10000 步，Warmup 设置 300 - 500 步即可。这能给 AdamW 优化器足够的时间去估计稳定的动量和方差。

</details>

### 3. 给你的实操避坑指南（针对 5M 数据混合）

由于你的数据集混合了不同领域（Chat / Safety / SWE / STEM / CoT 等），在跑这种大规模 LoRA 时极易出现以下问题：

1.  **数据混合比例 (Data Mixing Ratio)**
    *   不要直接简单粗暴地把 5M 拼在一起。STEM 和 CoT 数据通常较长且难，Chat 数据较短。如果随机组合，可能导致不同领域的拟合程度不一。
    *   **建议**：如果有条件，使用 Packing 技术（将多个短样本拼接成一个达到 `max_seq_length` 的长样本），这样不仅训练效率提升极大，也能均衡不同领域的学习。
2.  **Max Sequence Length 设置**
    *   因为有 Code 和 CoT，序列长度不能太短，建议至少设置 `max_length = 4096` 或 `8192`。
    *   结合大有效批次 (`eff_bs = 512`)，这需要极其庞大的吞吐能力，建议开启 FlashAttention-2 和 Gradient Checkpointing 以节省显存。
3.  **损失函数监控与早停 (Early Stopping)**
    *   因为你使用了 Rank 128 和全模块，如果数据集内部有部分质量较低的对话数据，模型可能会在这些数据上迅速过拟合。
    *   强烈建议在验证集中，**按领域划分不同的验证子集**（如单独算 Code 的 Loss，单独算 STEM 的 Loss）。如果发现整体 Loss 在降，但 Code 领域的 Loss 开始上升，说明出现了任务间冲突，可能需要提前停止，或者回去调整数据混合比例。
