# 处理类别不平衡方法

### 🧑‍💻 User

如何处理数据中的“类别不平衡”？例如二分类问题，训练集中80%分类为A，20%分类为B。

---

### 🤖 Assistant

在机器学习中，处理类别不平衡（Class Imbalance）是一个非常常见且关键的问题。在你提到的例子中（80%为类别A，20%为类别B），如果模型什么都不学，只盲目预测所有样本都是类别A，它的准确率（Accuracy）也能达到 $80\%$。这被称为**“准确率悖论”**。

为了让模型真正学会识别少数类（类别B），我们可以从**评估指标**、**数据层面**和**算法层面**进行处理。以下是解决该问题的核心策略：

### 1. 核心策略一：使用正确的评估指标（极其重要）
在不平衡数据中，绝对不能只看准确率（Accuracy）。你需要关注模型在少数类（类别B）上的表现。

*   **精确率 (Precision) 与 召回率 (Recall):** 
    对于类别B，精确率衡量“预测为B的样本中有多少真的是B” ($P = \frac{TP}{TP + FP}$)，召回率衡量“实际为B的样本中有多少被找出来了” ($R = \frac{TP}{TP + FN}$)。
*   **F1-Score:** 综合精确率和召回率的调和平均数，非常适合评估不平衡问题：
    $$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$
*   **PR-AUC (Precision-Recall Area Under Curve):** 相比于常见的 ROC-AUC，在严重不平衡（如 95% vs 5%）时，PR 曲线下的面积是更好的评估指标。

### 2. 核心策略二：数据层面的重采样 (Resampling)
通过改变训练集的分布，强行让模型看到更平衡的数据。**注意：重采样只能在训练集上进行，绝不能用于验证集或测试集。**

*   **欠采样 (Under-sampling):** 随机丢弃一部分类别A的数据，使其数量与类别B一致（变成 20% vs 20%）。
    *   *适用场景:* 数据量极其庞大时。
    *   *缺点:* 会丢失类别A的潜在重要信息。
*   **过采样 (Over-sampling):** 随机复制类别B的数据，使其数量与类别A一致（变成 80% vs 80%）。
    *   *缺点:* 简单的复制容易导致模型在类别B上过拟合。
*   **SMOTE (合成少数类过采样技术):** 这是目前最常用的方法。它不直接复制类别B，而是通过插值在现有的类别B样本之间生成**新的、合成的**数据点。

```python
# 使用 imbalanced-learn 库进行 SMOTE 处理的示例
from imblearn.over_sampling import SMOTE
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

smote = SMOTE(random_state=42)
# 只对训练集进行重采样
X_train_resampled, y_train_resampled = smote.fit_resample(X_train, y_train)
```

### 3. 核心策略三：算法层面的处理（代价敏感学习）
不改变数据，而是改变模型犯错时的“惩罚力度”。

*   **类别权重 (Class Weights):** 告诉模型，预测错类别B的代价比预测错类别A的代价更高。在 80/20 的比例下，我们可以赋予类别B 4倍于类别A的权重。
    算法的损失函数将被修改为：
    $$Loss = w_A \times Loss_A + w_B \times Loss_B$$
    大多数 Scikit-Learn 模型都支持 `class_weight='balanced'` 参数，它会自动根据数据比例计算权重（即 $w_A = \frac{100}{2 \times 80} = 0.625$, $w_B = \frac{100}{2 \times 20} = 2.5$）。

```python
# 在逻辑回归或随机森林等模型中使用 class_weight
from sklearn.ensemble import RandomForestClassifier

# 'balanced' 会自动增加少数类 B 的权重
model = RandomForestClassifier(class_weight='balanced', random_state=42)
model.fit(X_train, y_train)
```

### 4. 调整决策阈值 (Threshold Moving)
默认情况下，二分类模型的输出概率大于 $0.5$ 时预测为类别B。在不平衡数据中，模型对类别B的预测概率通常偏低。
你可以使用 `predict_proba()` 获取概率，并**手动降低阈值**（例如，只要概率 $> 0.3$ 就预测为B），以此来提高类别B的召回率。

### 5. 选择对不平衡数据鲁棒的模型
基于树的集成算法通常比线性模型表现更好：
*   **XGBoost / LightGBM:** 这些梯度提升树模型内部支持设置 `scale_pos_weight` 参数，专门用于处理正负样本不平衡。
*   **EasyEnsemble / BalancedRandomForest:** 这些是结合了欠采样和集成学习的专门算法。它们多次对多数类进行欠采样，并训练多个基模型，最后进行投票。

### 6. 视作异常检测 (Anomaly Detection)
如果类别不平衡极其严重（例如 99% vs 1%），传统的分类算法可能完全失效。此时可以转变思路，将多数类A视为“正常”，少数类B视为“异常”。
*   可以使用 **Isolation Forest (孤立森林)** 或 **One-Class SVM**，只用类别A的数据训练模型，让模型学习“正常”的数据分布。当类别B输入时，模型会将其标记为偏离分布的异常值。

### 💡 总结建议：你应该先尝试什么？
对于 **80% vs 20%** 这种中度不平衡，建议的解决路径如下：
1.  首先，放弃 Accuracy，改用 **F1-Score** 或 **PR-AUC** 评估模型。
2.  最优先尝试：在你的机器学习算法中开启 **`class_weight='balanced'`**。这种方法最简单，不需要修改数据，且往往效果很好。
3.  如果效果不佳，第二步尝试使用 **`SMOTE`** 对训练集进行过采样。
4.  根据业务需求，通过调整模型的预测概率**阈值**，在精确率和召回率之间找到最佳平衡点。
