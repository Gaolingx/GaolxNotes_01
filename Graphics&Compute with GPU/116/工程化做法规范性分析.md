# 工程化做法规范性分析

### 🧑‍💻 User

```
跨引擎/软件的灯光校准环境设置：

准备一张24色卡，并在场景中放置三个参考物体——透明光滑黑球、粗糙中灰球、镜面金属球。将相机曝光设为EV1，然后分别调整平行光和天光的颜色与强度，直至满足以下两个条件：

1. 在4000K的平行光照射下，其暖色调在直射区域正好被天光的冷色调完全中和，实现色彩平衡；
2. 此时对屏幕进行取色，色卡上的中灰色块以及中灰球表面的灰色，均与sRGB色彩空间的标准中灰（RGB 128,128,128）完全吻合。
```

这是目前比较规范的做法吗？

---

### 🤖 Assistant

**简单直接的回答：**

这段描述**并不符合**目前主流的“基于物理的渲染（PBR）”工业标准（如好莱坞影视特效或3A游戏标准流程）。

它更多属于**“资产审美/风格化（Lookdev）”**的一种特定约定或**老一代非PBR流程**的经验法则。虽然它能保证团队内部资产的一致性，但它违背了物理光照的基本原则，会导致资产在不同光照环境下表现不自然。

以下是这段描述中与现代标准PBR流程相悖的三个关键点分析：

### 1. 曝光值 EV1 的设置（严重偏离物理值）
*   **你的描述：** 将相机曝光设为 EV1。
*   **标准做法：** 在PBR流程中，如果是模拟室外日光环境，相机曝光值（EV @ ISO 100）通常应在 **14 到 15** 之间；如果是室内摄影棚光照，通常在 **8 到 10** 之间。
*   **为什么这是个问题：** EV1 是极暗环境（如月光下的野外或极其昏暗的室内）的曝光值。如果在 EV1 下通过调整灯光强度把场景照亮到正常亮度，意味着你的灯光强度数值极低（非物理数值）。这会导致当你把这个物体放入一个真实的 HDR 环境（比如强度为 100,000 Lux 的日光）时，物体会显得完全错误。

### 2. 强行锁定 sRGB (128, 128, 128) 的误区
*   **你的描述：** 色卡中灰及中灰球必须吻合 RGB (128, 128, 128)。
*   **标准做法：** 标准中灰卡（Middle Grey）的物理反射率是 **18%**（线性空间 Linear 0.18）。
    *   在标准的 Gamma 2.2 (sRGB) 转换下：$0.18^{(1/2.2)} \times 255 \approx 117$。
    *   **128** 实际上对应的线性反射率约为 **21.8%**（$(128/255)^{2.2} \approx 0.218$）。
*   **为什么这是个问题：** 强行对齐 128 是早期的“经验主义”，认为 128 是 0-255 的中间值（50% 亮度），符合人眼感知的中间灰。但在严谨的 ACES 或 Linear sRGB 工作流中，中灰球应该呈现为 **RGB 117-122** 左右。强行对齐 128 会导致整个场景的 Calibration **过曝**，或者让贴图绘制者画出的贴图固有色（Albedo）普遍偏暗。

### 3. 4000K 与“冷暖中和”的这种白平衡方式
*   **你的描述：** 4000K 平行光（暖）被天光（冷）中和，以此实现色彩平衡。
*   **标准做法：**
    *   **中性光照（Neutral Light）：** 工业标准通常使用 **D65 (6500K)** 作为标准白色参考，或者直接使用纯白光（6500K）进行资产审查，以确保没有色偏。
    *   **白平衡（White Balance）：** 我们通常调整相机的白平衡来抵消色温，而不是通过“混合两种颜色的光”来物理抵消。
*   **为什么这是个问题：** 4000K 是明显的暖光（下午或清晨的阳光）。如果你用天光去强行把 4000K “洗白”，说明你的天光必然是极高饱和度的蓝色，且强度极高。这创造了一个**非自然的光照比（Lighting Ratio）**。在这种环境下校准出来的材质，一旦放到正常的 6500K 日光下，可能会显得过蓝或过冷。

### 什么是目前“规范”的做法？

如果你的目标是跨引擎（如从 Substance Painter 到 UE5/Unity）保持一致，目前的规范做法是 **ACES 标准流程** 或 **中性灰 PBR 流程**：

#### 1. 环境设置
*   **HDRI：** 使用一张经过校准的、去色的（Desaturated）或色温约为 6500K 的全景图（Studio HDRI），而不是“暖光+冷天光”的混合。
*   **平行光：** 如果使用平行光，强度和开尔文色温应符合物理定义（例如：Intensity = 3.14 Lux 或对应的 Unitless 值，Color = Pure White 或 6500K）。

#### 2. 中灰校准 (The 0.18 Rule)
*   **参考物：** 一个反射率为 0.18 (Linear) 的漫反射球。
*   **目标：** 在不开启 Tone Mapping (色调映射) 的线性视图下，该球采样值为 **0.18**；或者在开启 ACES/sRGB 监视器视图下，该球采样值约为 **117-122**（取决于具体的 ODT/Gamma 曲线），**绝不是 128**。

#### 3. 验证方法 (Macbeth Chart)
*   使用标准 Macbeth 24色卡。
*   确保色卡上的色块在渲染窗口中的取色值（Linear Float），与标准色卡的 PBR 参考值（Albedo）一致。例如，红色块的 Linear 值约为 $0.21$，那渲染出来如果也是 $0.21$ 左右（在没有高光干扰的情况下），则光照强度是正确的。

### 总结

你提供的方案更像是一个**“美术资产审查（Lookdev）的特定风格化规范”**，旨在让美术人员在一个“看起来像白天但又很中性”的环境里画贴图，并确保所有人的亮度基准都在“128”这个直观的数值上。

*   **优点：** 简单，数值直观（128好记），适合非PBR项目或手绘风格项目。
*   **缺点：** 物理不正确，无法通用于真实的物理光照环境，不符合 ACES 流程。

如果你的项目追求写实（Photorealistic），建议**放弃**这个“128 + EV1”的方案，转向基于 **0.18中灰 + 物理曝光值** 的校准体系。
