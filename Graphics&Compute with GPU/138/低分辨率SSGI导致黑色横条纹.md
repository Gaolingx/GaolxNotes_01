# 低分辨率SSGI导致黑色横条纹

### 🧑‍💻 User

当ray marching based ssgi的rt分辨率低到某个阈值为什么会出现黑色的横条纹？

---

### 🤖 Assistant

在基于屏幕空间光线步进（Ray Marching based SSGI / SSRT）的全局光照渲染中，当你把光线追踪（RT）的运行分辨率降低到某个阈值（例如 1/4 或 1/16 屏幕分辨率）时，出现**黑色横条纹（Black Horizontal Stripes / Banding Artifacts）**是一个非常典型的图形学Artifact。

这种现象通常是由**屏幕空间坐标降采样带来的量化误差**、**光线步进算法（如 DDA 或 Hi-Z）在低分辨率下的步长失真**，以及**深度厚度启发式（Depth Thickness Heuristic）判定失败**共同导致的。

具体原因可以归结为以下几个核心方面：

### 1. DDA光线步进与像素量化误差（为什么是横条纹？）
在屏幕空间进行 Ray Marching 时，通常会使用类似 2D DDA（Digital Differential Analyzer）或 Bresenham 画线算法的思想在像素网格上“步进”。
* **低分辨率导致的大步长：** 当 RT 分辨率极低时，光线在低分辨率 Buffer 上每前进一步，相当于在全分辨率屏幕上跨越了多个像素。
* **水平/垂直主导轴（Major Axis）：** DDA 算法会根据光线的斜率将其分为“以 X 轴为主”或“以 Y 轴为主”。当光线在屏幕空间非常平缓（斜率很小）时，X 是主轴。在极低分辨率下，光线在 Y 方向上的步进会被严重**量化（截断或四舍五入）**。
* **条纹的形成：** 这种 Y 轴上的严重截断会导致光线在跨越某些扫描线（Scanlines）时发生集中的深度采样跳跃。一大批相邻像素的光线在同一个 Y 坐标上“错失”了相交目标，导致该区域没有 GI 贡献（即黑色 $0$ 值），从而形成视觉上明显的水平横条纹。

### 2. 深度/厚度判定（Z-Thickness）的失效（为什么是黑色？）
SSGI 无法知道物体的背面，因此必须假设屏幕上的每个像素都有一个“厚度”（Thickness）。如果光线穿过深度缓冲的背面距离大于这个厚度，则认为光线飞到了物体背后，判定为**未命中（Miss）**，返回黑色。
* 在低分辨率下，使用的深度缓冲（Depth Buffer）往往也是降采样的。降采样的深度图丢失了高频几何细节。
* 光线由于步长过大，很容易直接“穿透”那些在低分辨率下显得过于单薄的几何体，或者由于插值误差，导致计算出的深度差 $ \Delta Z $ 远远超过设定的 Thickness 阈值。
* 结果就是大面积的**False Miss（错误判定为未命中）**，原本应该有间接光反射的地方变成了毫无贡献的黑色。

### 3. 交错渲染（Interleaved / Checkerboard Rendering）的失败
为了性能，极低分辨率的 SSGI 经常配合**空间交错（Spatial Interleaved）**或**棋盘格（Checkerboard）**模式进行发射：
* 例如，第 $N$ 帧只渲染偶数行，第 $N+1$ 帧只渲染奇数行。
* 当分辨率降到极低阈值时，相邻行/列之间的几何差异变得巨大。如果在重建（Reconstruction / Upsampling）阶段，空间滤波器或时序滤波器（TAA/SVGF）的采样半径不足以覆盖这些巨大的空隙，或者历史重投影（Temporal Reprojection）因为低分辨率的运动向量（Motion Vectors）产生严重误差，就会导致那些“未发射光线”的行无法被正确填充，直接暴露为黑色横带。

### 4. Hi-Z (Hierarchical Z) 追踪的层级跨越错误
现代 SSGI 几乎都会使用 Hi-Z 缓冲来加速 Ray Marching。
* 在极低分辨率下启动 Hi-Z 追踪，意味着光线起始就处于 Hi-Z 的较高 Mipmap 层级（例如 Mip 2 或 Mip 3）。
* 由于高层级的深度是保守计算的（通常取 $2 \times 2$ 块中的 Min 或 Max Depth），当分辨率过低时，这些 $2 \times 2$ 或 $4 \times 4$ 的像素块（Tiles）在屏幕上变得非常巨大。
* 光线在跨越这些巨大的 Tile 边界时，极易发生错误的相交判定，而这些 Tile 的边界往往是水平和垂直对齐的，这进一步加剧了水平条纹或网格状黑块的出现。

要解决低分辨率 SSGI 的黑条纹问题，通常需要在渲染管线的多个阶段进行干预：

1. **引入蓝噪声抖动（Blue Noise Jittering）：** 
   不要让所有像素在同一位置发射光线。给光线的原点（Origin）、方向（Direction）和初始步长增加高频蓝噪声偏移。这会将结构化的“横条纹”打散成高频的“雪花噪点”（Noise），后续的降噪器（Denoiser）处理噪点比处理条纹容易得多。
2. **各向异性深度降采样（Anisotropic Depth Downsampling）：**
   在生成低分辨率深度图时，不仅保存 Min/Max 深度，还可以保存深度的梯度（Gradient）信息，防止在低分辨率下丢失纤细物体的深度连续性。
3. **改进联合双边上采样（Joint Bilateral Upsampling）：**
   使用全分辨率的深度和法线（Depth/Normal）来指导低分辨率 GI 结果的上采样。如果全分辨率深度和低分辨率深度的差异过大，拒绝该低分辨率采样，避免将错误的黑边放大。
4. **调整 Thickness 参数：**
   当 RT 分辨率降低时，需要动态放宽 Thickness 阈值（即 $Thickness_{lowres} > Thickness_{fullres}$），以补偿由于步长变大带来的穿透误差。
5. **强大的时空降噪器（SVGF / ReSTIR GI）：**
   使用更现代的降噪和重采样技术，通过积累多帧的历史数据来填补低分辨率单帧带来的巨大信息空洞。
