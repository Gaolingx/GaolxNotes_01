# 共享内存过多导致运行时错误

### 🧑‍💻 User

cuda中，如果申请的lds过多，会导致cuda runtime error吗？

---

### 🤖 Assistant

是的，在 CUDA 中申请过多的共享内存（Shared Memory，在 AMD 体系中通常称为 LDS - Local Data Share）确实会导致错误。但具体是触发**编译错误**还是**CUDA runtime error（运行时错误）**，取决于你申请共享内存的方式。

以下是具体的分类情况：

### 1. 动态申请共享内存（会导致 CUDA Runtime Error）
如果你使用的是**动态共享内存**，即在内核定义中使用 `extern __shared__`，并在内核启动时通过执行配置 `<<<grid, block, shared_mem_size>>>` 传入大小：

```cpp
extern __shared__ float s_data[];
// ...
// 如果 shared_mem_size 超过了硬件允许的单 Block 最大限制
myKernel<<<grid, block, shared_mem_size>>>(...);
```

在这种情况下，编译器无法在编译阶段知道你会申请多少内存。如果 `shared_mem_size` 超过了限制，内核将无法启动，并会抛出 **CUDA runtime error**。
*   通常返回的错误码是 `cudaErrorInvalidConfiguration` (Invalid configuration argument) 或 `cudaErrorLaunchFailure`。
*   **注意 48KB 默认限制：** 从 Volta 架构开始，很多 GPU 单个 Block 最大支持 96KB、163KB 甚至 227KB 的共享内存。但是，CUDA 默认单个 Block 动态申请的上限是 **48KB**。如果你动态申请超过 48KB 但没有提前配置，**也会直接导致 Runtime Error**。你需要使用以下代码解锁更高的限制：
    ```cpp
    cudaFuncSetAttribute(myKernel, cudaFuncAttributeMaxDynamicSharedMemorySize, desired_shared_mem_size);
    ```

### 2. 静态申请共享内存（会导致编译错误）
如果你在代码中直接写死了共享内存的大小（**静态共享内存**）：

```cpp
__shared__ float s_data[100000]; // 申请 400KB 内存，远超单 Block 限制
```

这种情况下，编译器（`nvcc` / `ptxas`）在编译阶段就能发现申请的内存超过了硬件架构的物理上限。此时**不会**发生 runtime error，而是直接**编译失败**，抛出类似如下的错误：
> `ptxas error : Entry function 'myKernel' uses too much shared data (0x61a80 bytes, 0xc000 max)`

### 3. 未超硬件上限，但导致 Occupancy（占用率）过低（无错误，但性能下降）
如果你申请的共享内存没有超过单个 Block 的绝对物理上限，内核可以成功启动（不会有 Runtime Error）。但是，共享内存是流多处理器（SM）上所有 Block 共享的资源。

如果单个 Block 申请了非常多的共享内存，会导致一个 SM 上能同时驻留的 Block 数量急剧减少。例如，一个 SM 总共有 100KB 共享内存，你的一个 Block 申请了 80KB，那么这个 SM 上最多只能同时跑 1 个 Block。这会导致 **Occupancy（占用率）极低**，无法有效隐藏访存延迟，从而导致程序性能大幅下降。

<details>
<summary><b>展开查看：各代 GPU 架构的共享内存上限</b></summary>

不同架构的 GPU 对单个 Block 的共享内存上限有不同的物理限制：

*   **Maxwell / Pascal (Compute Capability 5.x / 6.x):** 单个 Block 最大 48 KB。
*   **Volta (Compute Capability 7.0):** 单个 Block 最大 96 KB。
*   **Turing (Compute Capability 7.5):** 单个 Block 最大 64 KB。
*   **Ampere (Compute Capability 8.0 - A100):** 单个 Block 最大 163 KB。
*   **Ampere (Compute Capability 8.6 - RTX 30系列):** 单个 Block 最大 99 KB。
*   **Hopper (Compute Capability 9.0 - H100):** 单个 Block 最大 227 KB。

*注：即使硬件支持大于 48KB，动态分配超过 48KB 时仍必须调用 `cudaFuncSetAttribute`。*
</details>

### 总结建议
如果你遇到了与共享内存相关的 CUDA runtime error，请检查：
1. 内核启动时的第三个参数（动态共享内存字节数）是否计算错误，导致传入了一个极大的值。
2. 如果你需要申请超过 48KB 的动态共享内存，是否忘记调用 `cudaFuncSetAttribute` 提升上限。
3. 记得在内核启动后加上 `cudaGetLastError()` 和 `cudaDeviceSynchronize()` 来准确捕获并打印具体的错误信息。
